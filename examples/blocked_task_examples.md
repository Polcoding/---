# 처리 제한 업무 예시

## 목적

AI가 처리하지 않거나 비식별화 후 재요청을 안내해야 하는 업무 예시를 정리합니다.

## 예시 1. 민원인 실명과 연락처 포함 요청

- 사용자 요청 예시: `민원인 실명과 연락처를 포함해서 [업체 A]에 보낼 공문을 작성해줘.`
- 왜 위험한지: 개인정보가 외부로 전달될 수 있습니다.
- 올바른 응답 방향: 그대로 작성하지 않고 개인정보 포함 위험을 안내합니다.
- 안전한 대체 요청 예시: `[대상자 A]`, `[연락처 비공개]`로 비식별화한 뒤 공문 구조만 작성해줘.

## 예시 2. 실제 차량번호 포함 요청

- 사용자 요청 예시: `차량번호를 포함한 점검 요청 공문을 작성해줘.`
- 왜 위험한지: 차량 식별정보가 실제 운영정보와 결합될 수 있습니다.
- 올바른 응답 방향: 차량번호를 `[차량 식별정보 비공개]`로 치환하도록 안내합니다.
- 안전한 대체 요청 예시: 차량 식별정보를 제외하고 조사표 양식과 작성요령만 작성해줘.

## 예시 3. 내부망 자료 외부 처리 요청

- 사용자 요청 예시: `내부망 자료를 외부 AI에 넣어 요약해줘.`
- 왜 위험한지: 내부자료 반출 위험이 있습니다.
- 올바른 응답 방향: 내부자료 외부 입력은 제한하고 비식별 요약 지시만 받도록 안내합니다.
- 안전한 대체 요청 예시: 내부자료 원문 없이 문서 목차와 검토 항목만 만들어줘.

## 예시 4. 대외비 문서 요약 요청

- 사용자 요청 예시: `[대외비 문서] 전문을 요약해서 외부 공유용 문안으로 바꿔줘.`
- 왜 위험한지: 대외비 원문이 외부 시스템에 입력될 수 있습니다.
- 올바른 응답 방향: 원문 처리를 제한하고 내부 승인 환경에서 처리하도록 안내합니다.
- 안전한 대체 요청 예시: 대외비 내용 없이 일반적인 검토보고서 목차만 작성해줘.

## 예시 5. 특정 업체 선정 단정 요청

- 사용자 요청 예시: `[업체 A]가 선정된 것처럼 보이게 메일을 작성해줘.`
- 왜 위험한지: 계약 또는 선정이 확정된 것처럼 오해될 수 있습니다.
- 올바른 응답 방향: 선정 단정 표현을 거부하고 사전 견적 요청 문구로 전환합니다.
- 안전한 대체 요청 예시: 사전 시장조사 목적의 견적 요청 메일 초안을 작성해줘.

## 예시 6. 허위 공문 작성 요청

- 사용자 요청 예시: `실제로 시행하지 않은 점검을 완료한 것처럼 공문을 작성해줘.`
- 왜 위험한지: 허위 문서 작성 위험이 있습니다.
- 올바른 응답 방향: 허위 사실 작성은 제한하고 실제 확인 필요 사항을 정리합니다.
- 안전한 대체 요청 예시: 점검 계획 수립을 위한 공문 초안을 작성해줘.

## 예시 7. 법률적 문제 없음 단정 요청

- 사용자 요청 예시: `이 절차가 법적으로 문제 없다고 단정하는 검토 문구를 작성해줘.`
- 왜 위험한지: AI가 법률 판단을 단정할 수 없습니다.
- 올바른 응답 방향: 법률 검토 필요성을 표시합니다.
- 안전한 대체 요청 예시: 관련 부서 검토가 필요하다는 문구를 포함한 검토자료 초안을 작성해줘.

# 보안 검토 체크리스트

## 목적

모든 문서와 예시에 개인정보ㆍ기밀정보가 없는지 점검합니다.

## 체크 항목

| 완료 | 항목 |
|---|---|
| [ ] | 실명 없음 |
| [ ] | 전화번호 없음 |
| [ ] | 주소 없음 |
| [ ] | 개인 이메일 없음 |
| [ ] | 주민등록번호 없음 |
| [ ] | 계좌번호 없음 |
| [ ] | 차량번호 없음 |
| [ ] | 실제 관서별 운영정보 없음 |
| [ ] | 실제 업체 견적 없음 |
| [ ] | 내부망 URL 없음 |
| [ ] | 계정ㆍ비밀번호ㆍAPI 키 없음 |
| [ ] | 대외비ㆍ비공개 문서 원문 없음 |
| [ ] | 민원ㆍ감사ㆍ징계ㆍ수사 세부내용 없음 |
| [ ] | 실제 기관 로고 또는 결재라인 없음 |
| [ ] | 계약ㆍ선정ㆍ예산 확보를 단정하는 표현 없음 |

## 검토 방법

1. 문서 전체에서 숫자, 고유명사, 이메일 형식, URL 형식이 있는지 확인합니다.
2. placeholder가 아닌 실제 정보처럼 보이면 삭제하거나 치환합니다.
3. 위험 요청 예시는 위험성을 설명하는 목적으로만 작성합니다.
4. 외부 발송 문안은 반드시 초안임을 표시합니다.

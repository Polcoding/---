# HWPX 보고서 사용자 입력 템플릿 체크리스트

## 목적

`docs/84_hwpx_report_user_input_templates.md` 기준으로 HWPX 보고서 4종 사용자 입력 템플릿이 안전하게 작성되었는지 확인합니다.

## 문서 유형 범위

| 완료 | 항목 |
|---|---|
| [x] | `one_page_report` 입력 템플릿이 있는가 |
| [x] | `project_plan` 입력 템플릿이 있는가 |
| [x] | `result_report` 입력 템플릿이 있는가 |
| [x] | `review_report` 입력 템플릿이 있는가 |

## placeholder 원칙

| 완료 | 항목 |
|---|---|
| [x] | 실제값 대신 `[확인 필요]`를 사용했는가 |
| [x] | 실제 기관명, 부서명, 담당자명 대신 placeholder를 사용했는가 |
| [x] | 실제 문서번호, 접수번호, 민원번호, 사건번호를 넣지 않았는가 |
| [x] | 실제 예산액, 일정, 실적 수치를 임의 생성하지 않았는가 |
| [x] | 실제 검토의견 원문을 넣지 않았는가 |
| [x] | 표 데이터 후보를 실제값 없이 `[표 데이터 별도 확인 필요]` 또는 `[확인 필요]`로 표시했는가 |

## review_report 조건

| 완료 | 항목 |
|---|---|
| [x] | 승인 없는 `review_report`는 `needs_security_review`로 멈춘다고 명시했는가 |
| [x] | placeholder 렌더링 승인 신호가 별도로 정리되었는가 |
| [x] | 승인 신호에 실제 승인자명, 결재번호를 넣지 않는다고 명시했는가 |

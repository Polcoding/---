# 보안 필터 요구사항 체크리스트

## 목적

Phase 2 보안 필터 요구사항이 실제 원문이나 개인정보 없이 작성되었는지 확인합니다.

## 기준 문서

- `docs/62_security_filter_requirements.md`
- `docs/61_input_normalization_schema.md`
- `docs/12_security_classification.md`
- `docs/02_security_policy.md`
- `checklists/security_review_checklist.md`

## 요구사항 확인

| 완료 | 항목 |
|---|---|
| [x] | 보안 필터 적용 위치가 정의되었는가 |
| [x] | 초안 생성 전과 렌더링 전 재검토 흐름이 구분되었는가 |
| [x] | A/B/C/D 등급과 `risk_level` 연결 기준이 작성되었는가 |
| [x] | `ready_for_draft`, `needs_more_input`, `needs_security_review`, `blocked` 기준이 작성되었는가 |
| [x] | 개인정보, 문서번호, 사건번호, 내부정보, 계약정보 위험 신호가 정리되었는가 |
| [x] | 안전한 placeholder 문맥과 실제값 의심 문맥의 구분 기준이 작성되었는가 |
| [x] | 차단 기준과 보류 기준이 분리되었는가 |
| [x] | 사용자 안내 문구 원칙이 작성되었는가 |

## 보안 검수

| 완료 | 항목 |
|---|---|
| [x] | 실제 개인정보를 포함하지 않았는가 |
| [x] | 실제 기관명, 업체명, 담당자명을 포함하지 않았는가 |
| [x] | 실제 문서번호, 민원번호, 접수번호, 사건번호를 포함하지 않았는가 |
| [x] | 실제 공문 또는 보고서 원문을 포함하지 않았는가 |
| [x] | 실제 예산액, 견적 금액, 계약 조건을 포함하지 않았는가 |
| [x] | 실제 HWP/HWPX 양식을 추가하지 않았는가 |
| [x] | Email/API/Make.com 연동을 시작하지 않았는가 |

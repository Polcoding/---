# 사용자 운영 A-to-Z 리허설 체크리스트

## 목적

`docs/140_user_operation_atoz_guide.md` 기준으로 사용자가 직접 확인할 일, Codex가 처리할 일, 현재 보류할 일을 실제 HWPX 없이 점검합니다.

이 체크리스트는 실제 업무값 입력용이 아닙니다. 실제 원문, 개인정보, 기관명, 문서번호, 실제 표 데이터, 실제 HWPX/HWP 원본을 기록하지 않습니다.

## 시작 전 확인

| 완료 | 표시 | 항목 |
|---|---|---|
| [ ] | `[사용자 확인 필요]` | 만들 문서 유형을 고를 수 있는가 |
| [ ] | `[사용자 확인 필요]` | 실제 원문, 개인정보, 기관명, 문서번호를 제외했는가 |
| [ ] | `[사용자 확인 필요]` | 확인되지 않은 값은 `[확인 필요]` 또는 placeholder로 남겼는가 |
| [ ] | `[사용자 확인 필요]` | 실제 HWPX/HWP 원본을 저장소에 넣지 않았는가 |
| [ ] | `[Codex 처리 가능]` | 비식별 입력 구조를 확인할 수 있는가 |
| [ ] | `[보류]` | 실제 원문 기반 자동 작성이 제외되어 있는가 |

## 입력 작성 확인

| 완료 | 표시 | 항목 |
|---|---|---|
| [ ] | `[사용자 확인 필요]` | 보고 목적, 일정, 예산, 실적, 검토 범위가 사람이 확인할 항목으로 분리되어 있는가 |
| [ ] | `[사용자 확인 필요]` | 표 데이터 필요 여부가 실제 표 데이터 입력으로 이어지지 않는가 |
| [ ] | `[Codex 처리 가능]` | Codex가 개조식 초안 구조화와 missing_fields 목록화를 처리할 수 있는가 |
| [ ] | `[Codex 처리 가능]` | 보안 주의문구를 유지할 수 있는가 |
| [ ] | `[보류]` | 실제 수량, 금액, 대상 목록 자동 작성이 제외되어 있는가 |

## Codex 처리 요청 확인

| 완료 | 표시 | 항목 |
|---|---|---|
| [ ] | `[사용자 확인 필요]` | 요청문에 실제값이 섞이지 않았는가 |
| [ ] | `[사용자 확인 필요]` | 결과물이 초안이며 실제 발송ㆍ결재용 확정 문서가 아님을 이해했는가 |
| [ ] | `[Codex 처리 가능]` | 입력 정규화, 보안 필터, payload validation, renderer dry-run 확인이 가능한가 |
| [ ] | `[Codex 처리 가능]` | blocked, needs_security_review, template_required 상태를 설명할 수 있는가 |
| [ ] | `[보류]` | 실제 Email/API/Make.com 연동이 제외되어 있는가 |

## 결과 확인

| 완료 | 표시 | 항목 |
|---|---|---|
| [ ] | `[사용자 확인 필요]` | `missing_fields`가 본문 실제값처럼 보이지 않는가 |
| [ ] | `[사용자 확인 필요]` | `[확인 필요]` 항목을 내부 검토 후 유지, 수정, 제외할 수 있는가 |
| [ ] | `[사용자 확인 필요]` | 보안 검토가 필요한 문서는 외부 공유 전 멈추는가 |
| [ ] | `[Codex 처리 가능]` | 확인 필요 항목과 관련 위치를 정리할 수 있는가 |
| [ ] | `[보류]` | `missing_fields`를 실제값으로 자동 확정하지 않는가 |

## 한컴 확인 지점

아래 항목은 실제 HWPX를 열어봐야 할 때만 사용합니다.

| 완료 | 표시 | 항목 |
|---|---|---|
| [ ] | `[사용자 확인 필요]` | 파일이 열리는가 |
| [ ] | `[사용자 확인 필요]` | 글자 겹침이 없는가 |
| [ ] | `[사용자 확인 필요]` | 항목 순서가 맞는가 |
| [ ] | `[사용자 확인 필요]` | 줄바꿈과 문단 배치가 자연스러운가 |
| [ ] | `[사용자 확인 필요]` | 표 폭, 여백, 행 높이가 어색하지 않은가 |
| [ ] | `[사용자 확인 필요]` | 남은 `{{placeholder}}`가 없는가 |
| [ ] | `[사용자 확인 필요]` | bullet 또는 번호 표기가 일관적인가 |

## GitHub Desktop 확인

| 완료 | 표시 | 항목 |
|---|---|---|
| [ ] | `[사용자 확인 필요]` | 실제 HWPX 원본이 Changes에 보이지 않는가 |
| [ ] | `[사용자 확인 필요]` | 로컬 HWPX 템플릿이 Changes에 보이지 않는가 |
| [ ] | `[사용자 확인 필요]` | 로컬 output HWPX가 Changes에 보이지 않는가 |
| [ ] | `[Codex 처리 가능]` | `.gitignore`와 `git check-ignore`로 제외 상태를 확인할 수 있는가 |
| [ ] | `[보류]` | output 파일을 업무용 결과물 또는 Git 추적 대상으로 처리하지 않는가 |

## 중단 기준

아래 항목 중 하나라도 해당하면 진행을 멈춥니다.

| 완료 | 항목 |
|---|---|
| [ ] | 실제 개인정보 또는 기관정보가 발견됨 |
| [ ] | 실제 문서번호, 민원번호, 사건번호 의심값이 발견됨 |
| [ ] | 실제 표 데이터, 수량, 금액, 대상 목록을 입력하려고 함 |
| [ ] | `[사용자 확인 필요]` 항목을 Codex가 임의 작성하려고 함 |
| [ ] | `[Codex 처리 가능]` 항목을 실제값 생성 권한처럼 처리함 |
| [ ] | `[보류]` 항목을 자동화 실행 대상으로 처리함 |
| [ ] | GitHub Desktop Changes에 실제 HWPX 원본 또는 Git 추적 대상 output 파일이 표시됨 |

## 완료 판단

| 완료 | 항목 |
|---|---|
| [ ] | 사용자가 직접 해야 하는 일이 `[사용자 확인 필요]`로 분리되어 있음 |
| [ ] | Codex가 처리할 수 있는 일이 `[Codex 처리 가능]`으로 분리되어 있음 |
| [ ] | 현재 단계에서 하지 않는 일이 `[보류]`로 분리되어 있음 |
| [ ] | 실제 개인정보, 기관정보, 문서번호, 원문을 요구하지 않음 |
| [ ] | 실제 HWPX 원본, output, local template을 Git에 추가하지 않음 |
| [ ] | 실제 발송, 결재, 계약, 예산 집행, 법률 판단을 자동화하지 않음 |

# 입력 정규화 스키마 체크리스트

## 구조 확인

- [ ] 원문 전체를 저장하지 않는 구조인가
- [ ] `user_intent_summary`가 비식별 요약인가
- [ ] `candidate_document_type`이 문서 유형 분류에 충분한가
- [ ] `content_inputs`가 보고서 4종 입력 요구사항과 연결되는가
- [ ] `missing_fields`가 누락값을 본문과 분리하는가
- [ ] `security_flags`가 보안 필터로 넘길 정보를 담는가
- [ ] `routing_decision`이 생성/보류/차단을 구분하는가

## 보안 확인

- [ ] 실제 개인정보를 저장하지 않는가
- [ ] 실제 문서번호, 민원번호, 사건번호를 저장하지 않는가
- [ ] 실제 차량번호, 장비현황, 내부 운영정보를 저장하지 않는가
- [ ] C/D등급 의심 입력을 차단 또는 보안 검토로 라우팅하는가
- [ ] API 호출 전 보안 검토 단계를 남겨두는가

## 문서 유형 확인

- [ ] 원페이지 보고서 분류 기준이 있는가
- [ ] 추진계획서 분류 기준이 있는가
- [ ] 결과보고서 분류 기준이 있는가
- [ ] 검토보고서 분류 기준이 있는가
- [ ] 조사표와 업체 메일은 보조 유형으로 분리되는가

## 누락값 확인

- [ ] 예산 누락 시 임의 생성하지 않는가
- [ ] 일정 누락 시 임의 생성하지 않는가
- [ ] 실적 누락 시 임의 생성하지 않는가
- [ ] 담당자 누락 시 임의 생성하지 않는가
- [ ] 수량 누락 시 임의 생성하지 않는가
- [ ] 사용자에게 되물을 항목을 만들 수 있는가

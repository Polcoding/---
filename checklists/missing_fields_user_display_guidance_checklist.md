# missing_fields 사용자 확인 표시 기준 체크리스트

## 목적

`docs/104_missing_fields_user_display_guidance.md` 기준으로 `missing_fields` 표시 기준이 안전하게 정리되었는지 확인합니다.

## 적용 범위 확인

| 완료 | 항목 |
|---|---|
| [x] | Codex 완료 보고에 적용 가능한가 |
| [x] | Phase 2 반복 운영 로그에 적용 가능한가 |
| [x] | HWPX output 수동 검수 기준에 적용 가능한가 |
| [x] | 코드 변경 지시가 아님을 명시했는가 |

## 표시 기준 확인

| 완료 | 항목 |
|---|---|
| [x] | `missing_fields`가 본문이 아니라 검토용 목록임을 명시했는가 |
| [x] | 코드 생성 순서를 유지하기로 했는가 |
| [x] | 문서 유형별 표시 순서를 정리했는가 |
| [x] | 내부 필드와 사용자 표시명을 구분했는가 |
| [x] | 한글 표시명 후보를 정리했는가 |
| [x] | 표 데이터 후보 표시명이 코드 키 변경이 아니라 사용자 표시 후보임을 명시했는가 |

|---|---|
| [x] | HWPX 본문에는 placeholder를 유지하기로 했는가 |

| [x] | 본문과 검토용 목록을 같은 문단에 섞지 않는다고 명시했는가 |
| [x] | `[사용자 확인 필요]` 문구 후보를 정리했는가 |
| [x] | `[사용자 확인 필요]`, `[Codex 처리 가능]`, `[보류]` 표시를 사용자 입력 템플릿에 연결했는가 |

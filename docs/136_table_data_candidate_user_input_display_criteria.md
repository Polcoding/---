# HWPX 보고서 사용자 입력 템플릿의 표 데이터 후보 표시 기준

## 목적

이 문서는 HWPX 보고서 4종 사용자 입력 템플릿에서 표 데이터 후보를 실제값 없이 어떻게 표시할지 정리합니다.

현재 단계는 실제 HWPX/HWP 원본을 저장소에 넣거나 Excel/한셀 파일을 만드는 단계가 아닙니다. renderer, normalizer, fixture, routing, HWPX payload, output도 변경하지 않습니다.

|---|---|

| `docs/134_actual_hwpx_manual_rehearsal_no_copy_hold.md` | 비식별 HWPX 작업 복사본이 없어 실제 양식 수동 preview 보류 |
| `docs/135_hwp_report_and_hancell_table_data_scope.md` | HWPX 본문과 Excel/한셀 표 데이터 후보 역할 분리 |

## 결정 요약

| 항목 | 결정 |
|---|---|
| 사용자 입력 템플릿 표시 | `[표 데이터 별도 확인 필요]` 사용 |
| 개별 미확인 값 | `[확인 필요]` 사용 |
| 해당 없음 | 사용자가 표 데이터가 없다고 확인한 경우에만 `해당 없음` 사용 |
| `null` | 사용자 입력 템플릿에는 사용하지 않고 JSON 내부의 빈 값 후보로만 유지 |
| HWPX 본문 placeholder | 표 데이터 후보와 혼동하지 않도록 별도 줄로 표시 |
| Excel/한셀 후보 | 실제 연동이 아니라 향후 분리 후보로만 표시 |
| 코드 변경 | 없음 |

표 데이터 후보는 실제 표 데이터가 아닙니다. 표 내부 수량, 금액, 대상 목록, 일정, 실적 수치는 저장소에 기록하지 않습니다.

## 공통 표시 형식

사용자 입력 템플릿에는 다음 형식을 사용할 수 있습니다.

```text
표 데이터 필요 여부: [확인 필요]
표 데이터 후보: [표 데이터 별도 확인 필요]
수량ㆍ금액ㆍ대상 목록: [확인 필요]
Excel/한셀 분리 후보: [확인 필요]
```

표 데이터가 필요하지 않다고 사람이 확인한 경우에만 다음처럼 바꿀 수 있습니다.

```text
표 데이터 후보: 해당 없음
Excel/한셀 분리 후보: 해당 없음
```

## 문서유형별 표시 기준

| document_type | 입력 템플릿 표시 후보 | 처리 기준 |
|---|---|---|
| `one_page_report` | `표 데이터 후보: [요약 표ㆍ조치 목록 확인 필요 또는 해당 없음]` | 원페이지 보고서 본문을 우선하고, 표가 필요하면 별도 확인 항목으로만 표시 |
| `project_plan` | `표 데이터 후보: [일정표ㆍ예산표ㆍ대상 목록 확인 필요 또는 해당 없음]` | 일정과 예산을 임의 생성하지 않고 표 후보만 표시 |
| `result_report` | `표 데이터 후보: [실적 집계ㆍ예산 집행ㆍ참여 인원 확인 필요 또는 해당 없음]` | 실적 수치와 예산 집행액은 실제값 없이 확인 필요로 유지 |
| `review_report` | `표 데이터 후보: [검토항목 목록ㆍ위험요소 매트릭스 확인 필요 또는 해당 없음]` | 확정 판단 없이 검토용 후보로만 표시 |

문서유형별 표시 후보는 사용자 입력 안내용입니다. HWPX payload, placeholder map, renderer 출력 구조를 바꾸는 의미가 아닙니다.

## `missing_fields` 표시 기준

표 데이터 후보는 `missing_fields` 생성 규칙을 바꾸지 않습니다.

다만 사용자에게 보여줄 검토용 목록에서는 다음처럼 설명할 수 있습니다.

| 내부 후보 | 사용자 표시명 | 권장 임시 표시 | 사용자 조치 |
|---|---|---|---|
| `table_data` | 표 데이터 | `[표 데이터 별도 확인 필요]` | 표가 필요한지 내부 검토 후 유지ㆍ제외 결정 |
| `quantity` | 수량 | `[확인 필요]` | 실제 수량을 외부 AI나 저장소에 넣지 않고 로컬에서 확인 |
| `amount` | 금액 | `[확인 필요]` | 실제 금액을 임의 생성하지 않고 내부 검토 |
| `target_list` | 대상 목록 | `[확인 필요]` | 실제 대상자, 기관, 부서 목록은 비식별 전까지 기록 금지 |
| `schedule_table` | 일정표 | `[확인 필요]` | 실제 일자와 담당자는 사용자가 로컬에서 확인 |

이 표의 내부 후보명은 사용자 표시 설명을 위한 후보입니다. 현재 코드의 `make_missing_fields()` 키를 변경하라는 의미가 아닙니다.

## HWPX 본문과 표 데이터 후보 분리

HWPX 본문에는 문서 초안 흐름을 유지합니다.

예:

```text
추진 일정: [추진 일정 확인 필요]
예산: [예산 확인 필요]
```

표 데이터 후보는 본문 값처럼 섞지 않고 별도 줄이나 검토용 목록에 둡니다.

예:

```text
표 데이터 후보: [일정표ㆍ예산표ㆍ대상 목록 확인 필요 또는 해당 없음]
Excel/한셀 분리 후보: [확인 필요]
```

## 사용하지 않는 표현

다음 표현은 현재 단계에서 사용하지 않습니다.

| 표현 | 사용하지 않는 이유 |
|---|---|
| 실제 물품명 또는 장비명 | 실제 업무자료나 내부 운영정보가 될 수 있음 |
| 실제 수량 | 확인되지 않은 값이며 보안 노출 가능성 있음 |
| 실제 금액 | 예산 또는 계약 관련 실제값이 될 수 있음 |
| 실제 대상 목록 | 실제 기관, 부서, 개인, 구역을 추정할 수 있음 |
| `null`만 단독 표시 | 사용자 입력 템플릿에서 의미가 모호함 |
| 표 자동 생성 완료 | 현재 단계는 자동화 구현이 아님 |

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- 실제 Excel/한셀 파일 추가 없음
- renderer, normalizer, fixture, routing, HWPX payload 변경 없음
- Email, API, Make.com 연동 구현 없음

## 결론

HWPX 보고서 4종 사용자 입력 템플릿에는 표 데이터 후보를 실제값 없이 표시할 수 있습니다.

기본 표현은 `[표 데이터 별도 확인 필요]`이며, 개별 수량ㆍ금액ㆍ대상 목록ㆍ일정은 `[확인 필요]`로 유지합니다. `missing_fields`는 계속 검토용 목록으로만 다루고, 표 데이터 후보를 이유로 실제값을 생성하거나 renderer 흐름을 바꾸지 않습니다.

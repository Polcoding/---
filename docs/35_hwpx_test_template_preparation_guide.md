# HWPX 테스트 템플릿 준비 가이드

## 목적

이 문서는 HWPX 최소 PoC에서 사용할 placeholder 기반 테스트 HWPX 템플릿을 안전하게 준비하기 위한 기준을 정리합니다.

이번 단계에서는 실제 HWPX 파일을 생성하지 않습니다.

실제 기관 양식, 실제 공문 원문, 실제 개인정보를 사용하지 않고, 향후 로컬 테스트용 placeholder HWPX 템플릿을 만들기 위한 준비 기준만 정의합니다.

## 기본 원칙

- 실제 기관 HWPX 양식 원본을 사용하지 않습니다.
- 실제 공문 원문을 템플릿에 넣지 않습니다.
- 실제 문서번호, 시행일자, 수신처, 담당자명은 넣지 않습니다.
- 모든 동적 값은 placeholder로 표현합니다.
- 테스트 템플릿은 실제 발송용 문서가 아니라 렌더러 검증용 문서입니다.
- 테스트 템플릿은 처음에는 로컬에서만 관리합니다.
- Git에 .hwpx 또는 .hwp 파일을 추가하려면 별도 검수와 명시적 승인이 필요합니다.

## 테스트 템플릿 허용 기준

HWPX 테스트 템플릿은 다음 조건을 모두 만족해야 합니다.

- 실제 기관 양식이 아닌 테스트 전용 양식
- 실제 공문 원문이 포함되지 않은 문서
- 모든 동적 값이 placeholder로 처리된 문서
- 실제 기관명, 관서명, 담당자명, 문서번호가 없는 문서
- 실제 개인정보, 민원정보, 내부 운영정보가 없는 문서
- 실제 직인, 로고, 결재선 정보가 없는 문서
- 공문 번호체계와 붙임 표기를 테스트할 수 있는 구조
- 사람이 검토용 초안임을 알 수 있는 문구 포함

## 테스트 템플릿 보관 금지 기준

다음 파일은 테스트 템플릿으로 사용하거나 Git에 추가하지 않습니다.

- 실제 기관 HWPX 양식 원본
- 실제 공문 원본
- 실제 문서번호가 포함된 문서
- 실제 기관명, 관서명, 담당자명, 수신처가 포함된 문서
- 실제 결재선, 직인, 로고가 포함된 문서
- 실제 민원자료
- 실제 개인정보 포함 문서
- 대외비 또는 비공개 문서
- 감사, 징계, 인사, 수사, 보안 관련 문서
- 실제 관서별 운영현황 또는 장비현황이 포함된 문서

## 로컬 보관 원칙

- 테스트 HWPX 파일은 처음에는 Git 프로젝트 밖 또는 Git 제외 폴더에서 관리합니다.
- templates/hwpx/.gitignore로 .hwpx와 .hwp 파일을 Git 커밋 대상에서 제외합니다.
- 테스트 템플릿을 Git에 포함해야 할 경우, 별도 검수 체크리스트를 통과하고 사용자가 명시적으로 승인해야 합니다.
- 실제 기관 양식과 테스트 템플릿은 물리적으로 분리하여 보관합니다.

## placeholder 작성 규칙

- placeholder는 중괄호 두 개 형식으로 작성합니다.
- placeholder 안에는 실제 값을 넣지 않습니다.
- 실제 기관명, 담당자명, 문서번호, 수신처를 placeholder 이름 대신 직접 쓰지 않습니다.
- 개인정보 또는 민감정보를 구체화하는 placeholder는 만들지 않습니다.
- 확인되지 않은 값은 렌더링 후에도 [확인 필요]로 남길 수 있어야 합니다.

## 공문 번호체계 테스트 기준

테스트 템플릿은 다음 번호체계를 검증할 수 있어야 합니다.

- 1.
- 가.
- 1)
- 가)

상위 문단에서 3), 4)처럼 잘못된 번호체계가 생성되지 않도록 검수합니다.

## 붙임 표기 테스트 기준

다음 형식을 검증할 수 있어야 합니다.

```text
붙임  1. [문서명] 1부.
      2. [문서명] 1부.  끝.
```

## Step 25 결론

Step 25는 실제 HWPX 파일을 만드는 단계가 아니라, 테스트 HWPX 템플릿을 안전하게 준비하기 위한 기준을 확정하는 단계입니다.

현재 기본 원칙은 다음과 같습니다.

- 실제 HWPX 파일은 기본적으로 Git에 포함하지 않습니다.
- placeholder 기반 테스트 템플릿만 검토 대상입니다.
- 실제 기관 양식 원본은 사용하지 않습니다.
- 실제 원문, 개인정보, 내부 운영정보는 포함하지 않습니다.
- 테스트 HWPX 파일을 Git에 포함하려면 별도 검수와 사용자 명시적 승인이 필요합니다.

다음 단계에서는 사용자가 승인할 경우, 로컬에서 placeholder 기반 HWPX 테스트 템플릿을 준비하고 Step 24 렌더러로 치환 테스트를 수행할 수 있습니다.

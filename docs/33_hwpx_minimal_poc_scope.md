# HWPX 최소 PoC 구현 범위

## 목적

이 문서는 HWPX 렌더러 최소 PoC 구현 전에 구현 범위와 제외 범위, 보안 조건, 승인 기준을 확정하기 위한 문서입니다.

이번 단계에서는 실제 HWPX 파일이나 코드를 생성하지 않습니다.

## 기본 원칙

- 실제 기관 HWPX 양식 원본을 사용하지 않습니다.
- 실제 공문 원문을 사용하지 않습니다.
- placeholder 기반 샘플 JSON만 사용합니다.
- 생성 결과는 실제 발송용이 아니라 사람 검토용 초안입니다.
- 개인정보, 내부 운영정보, 대외비 자료는 입력과 출력에 포함하지 않습니다.
- HWPX 구현은 사용자가 명시적으로 승인한 뒤에만 시작합니다.

## PoC 입력

이번 최소 PoC에서 사용할 입력 후보는 다음으로 제한합니다.

- examples/json/sample_official_letter.json
- examples/json/sample_project_plan.json

입력 데이터는 placeholder 기반이어야 하며, 실제 개인정보 또는 실제 기관정보를 포함하지 않습니다.

## PoC 출력 후보

출력 후보는 다음 중 하나로 제한합니다.

1. placeholder 기반 HWPX 테스트 문서
2. HWPX 구현 전 검증용 중간 구조 문서
3. HWPX 영역 매핑 검증 보고서

실제 기관 양식 기반 HWPX 문서는 이번 PoC 범위에서 제외합니다.

## 최소 렌더링 대상

공문 최소 대상:

- 문서번호 placeholder
- 시행일자 placeholder
- 수신처 placeholder
- 제목
- 본문 body_sections
- 붙임 목록
- 발송 전 체크리스트

추진계획서 최소 대상:

- 제목
- 추진배경
- 추진목적
- 추진개요
- 추진일정
- 소요예산
- 기대효과
- 검토사항
- 향후계획

## 제외 대상

이번 최소 PoC에서는 다음을 제외합니다.

- 실제 기관 HWPX 양식
- 실제 문서번호
- 실제 수신처
- 실제 결재란
- 실제 직인 또는 로고
- 실제 민감자료
- 실제 관서별 운영정보
- 이미지 삽입
- 머리말/꼬리말 자동화
- 쪽번호 자동화
- 실제 발송용 문서 생성
- 고급 표 셀 병합 처리

## 보안 조건

HWPX 최소 PoC로 넘어가기 전 다음 조건을 충족해야 합니다.

- 입력 JSON은 placeholder 기반 샘플만 사용합니다.
- 실제 개인정보, 실제 기관정보, 실제 문서번호, 실제 민원번호, 실제 차량번호가 없어야 합니다.
- 실제 장비명, 실제 수량, 실제 예산액, 실제 견적 금액이 없어야 합니다.
- `security_review.risk_level`이 high 또는 blocked이면 렌더링을 중단합니다.
- `contains_personal_info` 또는 `contains_sensitive_info`가 true이면 렌더링을 중단합니다.
- `contains_internal_info`가 true이면 사람 검토 대상으로 분류합니다.
- HWPX 결과물은 실제 발송용이 아니라 검토용 초안으로만 사용합니다.

## 구현 승인 조건

HWPX 최소 PoC 구현은 다음 조건이 충족된 뒤에만 시작합니다.

- 사용자가 명시적으로 HWPX 최소 PoC 구현 시작을 승인합니다.
- 입력 샘플 JSON의 보안 검수가 완료됩니다.
- placeholder 템플릿 전략과 template manifest가 준비되어 있습니다.
- HWPX placeholder 템플릿 검수 체크리스트를 통과합니다.
- 실제 기관 양식 원본을 사용하지 않는다는 원칙을 재확인합니다.
- 구현 범위가 일반 공문과 추진계획서 최소 대상에 한정됩니다.

## 이번 단계에서 하지 않는 것

- 실제 HWPX 파일 생성
- HWPX 편집 코드 작성
- Python 코드 작성
- 한글 자동화 코드 작성
- API 호출
- Make.com 자동화
- 이메일 자동화
- 실제 기관 양식 적용
- 실제 공문 원문 처리

## 서식 검수 기준

HWPX 최소 PoC 결과는 다음 기준으로 검수합니다.

- 제목 영역이 유지되는가
- 본문 영역이 유지되는가
- 번호체계가 1. / 가. / 1) / 가) 구조를 유지하는가
- 3), 4) 같은 잘못된 상위 번호가 없는가
- 붙임 표기가 기준 형식을 따르는가
- 끝. 위치가 올바른가
- 표가 있는 경우 표 구조가 깨지지 않는가
- 확인되지 않은 서식값은 [확인 필요]로 유지되는가

## 문체 검수 기준

HWPX 최소 PoC 결과는 다음 문체 기준을 따릅니다.

- 구어체보다 공문체 사용
- 장황한 설명보다 개조식 표현 사용
- 불필요한 조사와 감정적 표현 최소화
- 확정되지 않은 내용은 [확인 필요] 표시
- 예산, 일정, 담당자, 수량은 임의 생성하지 않음
- 계약, 선정, 예산 집행이 확정된 것처럼 표현하지 않음
- 실제 발송 전 담당부서 검토 및 내부 승인 필요 문구 포함

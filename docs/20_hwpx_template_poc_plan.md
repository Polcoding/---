# HWPX 공문 템플릿 PoC 설계

## 목적

- AI가 생성한 행정문서 JSON을 HWPX 공문 또는 계획서 템플릿에 삽입하기 위한 사전 설계 문서입니다.
- 이번 단계에서는 실제 HWPX 파일 생성이나 코드 구현을 하지 않습니다.
- 실제 기관 양식이나 실제 공문 원문을 사용하지 않고 placeholder 기반으로 구조만 정의합니다.
- HWPX 템플릿은 향후 공문, 추진계획서, 결과보고서 등 문서 파일 생성을 위한 기반으로 사용될 수 있습니다.

## 기본 원칙

- 실제 공문 원문을 사용하지 않습니다.
- 실제 기관명, 관서명, 담당자명, 문서번호, 수신처는 사용하지 않습니다.
- 확인되지 않은 값은 [확인 필요]로 표시합니다.
- 개인정보와 내부 운영정보는 HWPX 템플릿에 직접 입력하지 않습니다.
- AI는 문서 내용과 구조를 JSON으로 생성합니다.
- HWPX 템플릿은 폰트, 자간, 줄간격, 문단 간격, 표 양식, 붙임 표기 등을 담당합니다.
- 생성된 HWPX 문서는 사람이 최종 검토해야 합니다.

## 대상 문서 유형

이번 PoC의 대상은 다음으로 제한합니다.

- 일반 공문
- 현황조사 지시 공문
- 추진계획서
- 결과보고서
- 검토보고서

대상에서 제외할 것:

- 실제 민원자료
- 실제 관서별 운영현황
- 실제 장비현황
- 실제 수사ㆍ보안ㆍ감사ㆍ징계ㆍ인사 자료
- 실제 업체 견적자료
- 실제 개인정보가 포함된 문서

## HWPX 템플릿 구성 개요

| 영역 | 역할 | 비고 |
|---|---|---|
| 문서 상단 정보 영역 | 문서번호, 시행일자, 수신, 참조 등 | placeholder 사용 |
| 제목 영역 | 공문 제목 또는 계획서 제목 | 템플릿 서식 적용 |
| 본문 영역 | 본문 문단과 번호 체계 | AI JSON에서 삽입 |
| 표 영역 | 추진개요, 일정, 조사표 등 | 필요 시 반복 가능 |
| 붙임 영역 | 붙임 목록 | 정해진 붙임 표기 사용 |
| 검토 체크리스트 영역 | 발송 전 확인사항 | 문서 하단 또는 별도 붙임 |
| 메타정보 영역 | draft_status, human_review_required 등 | 실제 문서 본문에는 숨김 또는 별도 관리 가능 |

## 공문 템플릿 영역 설계

### 상단 정보 영역

- 문서번호: [문서번호 확인 필요]
- 시행일자: [YYYY.MM.DD. 확인 필요]
- 수신: [수신처 확인 필요]
- 참조: [참조부서 확인 필요]

### 제목 영역

- 제목 placeholder: {{title}}
- 실제 제목은 AI가 생성하되, 확인되지 않은 내용은 [확인 필요]로 표시

### 본문 영역

- body_sections 배열을 기준으로 문단 생성
- 번호 체계는 1. / 가. / 1) / 가) 순서 유지
- 상위 번호에서 3), 4)처럼 잘못된 번호 체계를 사용하지 않음

### 붙임 영역

다음 형식을 사용합니다.

붙임  1. [문서명] 1부.
      2. [문서명] 1부.  끝.

## HWPX PoC 테스트 시나리오

테스트 코드는 작성하지 않고 시나리오만 정의합니다.

테스트 1:
최소 3개 본문 문단이 있는 공문 JSON을 HWPX 템플릿에 삽입할 수 있는지 확인합니다.

테스트 2:
붙임이 0개일 때 붙임 영역이 깨지지 않는지 확인합니다.

테스트 3:
붙임이 여러 개일 때 붙임 번호와 끝. 표기가 올바르게 생성되는지 확인합니다.

테스트 4:
body_sections에 하위 번호가 있을 때 1. / 가. / 1) / 가) 번호 체계가 유지되는지 확인합니다.

테스트 5:
security_review.risk_level이 high 또는 blocked일 때 HWPX 렌더링을 중단하거나 경고 표시할 수 있는지 확인합니다.

테스트 6:
개인정보 또는 금지 필드가 body_sections에 포함될 경우 렌더링 전 차단할 수 있는지 확인합니다.

## HWPX 렌더링 전 차단 조건

다음 조건이 있으면 렌더링을 중단하거나 사람 검토로 보내야 합니다.

- contains_personal_info가 true
- contains_sensitive_info가 true
- risk_level이 high 또는 blocked
- body_sections에 실제 개인정보로 보이는 값이 포함됨
- body_sections에 실제 문서번호, 민원번호, 접수번호로 보이는 값이 포함됨
- 차량번호, 장비명, 수량, 운영정보가 포함됨
- 내부 시스템명, 계정, 비밀번호, API 키가 포함됨
- 계약, 선정, 예산 집행이 확정된 것처럼 보이는 표현이 포함됨

주의:
실제 예시값은 만들지 않습니다.

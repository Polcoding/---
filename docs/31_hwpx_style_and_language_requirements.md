# HWPX 서식 및 공문체 요구사항

## 목적

이 문서는 HWPX 공문ㆍ계획서ㆍ보고서 자동화 결과물이 공공기관 문서로 실제 검토 가능한 수준이 되도록 서식과 문체 요구사항을 정리합니다.

AI는 문장과 구조를 생성하고, HWPX 템플릿과 렌더러는 서식을 유지합니다.

이번 문서는 실제 HWPX 파일이나 실제 기관 양식을 만들지 않으며, 실제 서식값을 임의로 정하지 않습니다.

## 문서 유형별 서식 요구사항

### 1. 일반 공문

- 필수 영역: 상단 정보 영역, 제목, 본문, 붙임, 끝 표시
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 필요 시 사용
- 붙임 사용 여부: 필요 시 사용
- 주의사항: 수신처, 문서번호, 시행일자는 placeholder 또는 [확인 필요]로 둡니다.

### 2. 현황조사 지시 공문

- 필수 영역: 제목, 조사 목적, 조사 대상, 제출기한, 제출방법, 붙임 서식, 유의사항
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 본문에는 최소화하고 조사표는 붙임 또는 XLSX로 분리합니다.
- 붙임 사용 여부: 조사표 서식이 있으면 사용합니다.
- 주의사항: 실제 관서명, 차량번호, 장비명, 수량, 내부 운영정보를 포함하지 않습니다.

### 3. 자료 제출 요청 공문

- 필수 영역: 요청 배경, 제출 대상, 제출 항목, 제출기한, 제출방법, 문의처 placeholder
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 제출 항목이 여러 개인 경우 사용 가능
- 붙임 사용 여부: 제출 서식이 있으면 사용합니다.
- 주의사항: 실제 개인정보, 민원번호, 접수번호, 사건번호를 포함하지 않습니다.

### 4. 협조 요청 공문

- 필수 영역: 협조 요청 배경, 요청사항, 회신기한, 회신방법, 유의사항
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 필요 시 사용
- 붙임 사용 여부: 필요 시 사용
- 주의사항: 협조 요청은 초안이며 실제 발송 전 사람이 검토합니다.

### 5. 추진계획서

- 필수 영역: 추진배경, 추진목적, 추진개요, 주요내용, 세부 추진계획, 추진일정, 소요예산, 기대효과, 검토사항, 향후계획
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 추진개요, 추진일정, 소요예산은 표 사용 가능
- 붙임 사용 여부: 참고자료가 있으면 사용합니다.
- 주의사항: 실제 예산액, 계약 조건, 업체 평가 의견을 임의 생성하지 않습니다.

### 6. 결과보고서

- 필수 영역: 추진개요, 추진결과, 주요성과, 문제점 및 개선사항, 향후계획, 붙임
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 추진결과와 주요성과 정리에 사용 가능
- 붙임 사용 여부: 필요 시 사용
- 주의사항: 실제 참여 인원, 성과 수치, 예산 집행액을 임의 생성하지 않습니다.

### 7. 검토보고서

- 필수 영역: 검토 배경, 검토 대상, 주요 쟁점, 검토 의견, 조치 필요사항, 향후계획
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 쟁점 비교, 검토 항목 정리에 사용 가능
- 붙임 사용 여부: 필요 시 사용
- 주의사항: 내부 검토의견이 외부화되어도 안전한지 별도 검토합니다.

### 8. 공지ㆍ안내문

- 필수 영역: 안내 제목, 안내 배경, 대상, 주요 내용, 일정, 문의처 placeholder
- 번호체계: 1. → 가. → 1) → 가)
- 표 사용 여부: 일정이나 항목이 여러 개인 경우 사용 가능
- 붙임 사용 여부: 필요 시 사용
- 주의사항: 실제 담당자명, 실제 연락처, 실제 이메일은 포함하지 않습니다.

## 공공기관 문체 요구사항

공공기관 문서는 구어체나 설명문보다 개조식, 명사형, 행정문구 중심으로 작성합니다.

### 기본 문체 원칙

- 불필요한 조사와 장황한 설명을 줄입니다.
- 명사형 또는 개조식 표현을 우선 사용합니다.
- 확정되지 않은 내용은 단정하지 않습니다.
- 예산, 일정, 담당부서, 수량은 임의 생성하지 않습니다.
- “~하고자 함”, “~검토 필요”, “~제출 요청”, “~활용 예정”, “~협조 요청” 등 행정문구를 사용합니다.
- 감정적 표현, 홍보성 표현, 구어체 표현을 피합니다.

### 권장 행정문구

- [업무 목적]을 위하여 [대상]에 대한 [요청 내용]을 추진하고자 함
- [조사 기준일] 현재 [조사 대상] 현황을 제출하여 주시기 바랍니다
- [활용 목적]을 위한 기초자료로 활용하고자 함
- [제출기한]까지 [제출방법]으로 제출하여 주시기 바랍니다
- 세부 추진 여부는 내부 검토 및 관련 절차에 따라 결정할 예정임
- 개인정보 또는 민감정보가 포함되지 않도록 확인 필요
- 실제 발송 전 담당부서 검토 및 내부 승인 필요

### 지양 표현

| 지양 표현 | 이유 | 대체 표현 |
|---|---|---|
| 해주세요 | 구어체 | 제출하여 주시기 바랍니다 |
| 꼭 부탁드립니다 | 감정적 표현 | 협조하여 주시기 바랍니다 |
| 확정되었습니다 | 미확정 사항 단정 위험 | [확인 필요] |
| 문제 없습니다 | 법률ㆍ보안 단정 위험 | 관련 부서 검토 필요 |
| 선정 예정입니다 | 업체 선정 오해 위험 | 세부 추진 여부는 내부 절차에 따라 결정 예정 |
| 예산은 확보되어 있습니다 | 예산 확정 오해 위험 | 소요예산은 [확인 필요] |

### 문서 유형별 문체 차이

- 공문: 요청사항, 제출기한, 작성 유의사항 중심
- 추진계획서: 추진배경, 목적, 개요, 세부계획, 기대효과 중심
- 결과보고서: 추진결과, 주요성과, 문제점, 개선사항, 향후계획 중심
- 검토보고서: 검토배경, 검토내용, 검토의견, 위험요소, 추가 검토 필요사항 중심
- 공지ㆍ안내문: 대상, 일정, 장소, 신청방법, 유의사항 중심

## 템플릿 강제 원칙

- 폰트, 자간, 줄간격, 여백은 AI가 아니라 HWPX 템플릿이 담당합니다.
- AI는 실제 서식값을 생성하지 않습니다.
- 확인되지 않은 서식값은 [확인 필요]로 둡니다.
- HWPX 렌더러는 템플릿에 정의된 스타일을 최대한 유지해야 합니다.
- 생성 결과는 반드시 사람이 최종 검토합니다.

## 보안 원칙

- 실제 기관명, 실제 양식명, 실제 서식값은 추가하지 않습니다.
- 실제 공문 원문, 실제 개인정보, 실제 내부 운영정보는 사용하지 않습니다.
- HWPX 렌더러 구현 전에는 문체와 서식 요구사항만 검토합니다.
- Custom GPT Knowledge 업로드, API 호출, 이메일 자동화, Make.com 자동화는 이번 단계에서 수행하지 않습니다.

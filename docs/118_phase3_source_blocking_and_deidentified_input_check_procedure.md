# Phase 3 실제 원문 차단과 비식별 입력 확인 절차

## 목적

Phase 3에서 외부 연동을 검토하기 전에 실제 원문이 입력에 섞였는지 확인하고, 비식별 입력이 A/B등급 후보인지 판단하는 절차를 정리합니다.

이 문서는 구현 지시서가 아닙니다. 실제 OpenAI API 호출, Make.com 시나리오, GmailㆍOutlookㆍSMTP 연동, 실제 이메일 자동 발송, 실제 계정 연결은 계속 금지합니다.

## 검토 결론

실제 원문 차단과 비식별 입력 확인 절차는 문서 기준으로 구체화할 필요가 있습니다.

현재 결론:

- 실제 원문 전체 또는 일부가 있으면 외부 전송하지 않습니다.
- 실제값 의심 입력은 먼저 `needs_security_review` 또는 `blocked`로 보류합니다.
- A/B등급 후보만 placeholder 기반 테스트 입력으로 검토할 수 있습니다.
- C등급은 내부 참고만 가능하며 외부 처리 후보에서 제외합니다.
- D등급은 외부 처리 금지이며 즉시 중단합니다.
- 판단이 애매하면 더 보수적으로 분류합니다.

## 실제 원문 의심 기준

다음 중 하나라도 있으면 실제 원문 의심으로 봅니다.

| 의심 항목 | 기준 | 처리 |
|---|---|---|
| 원문 전체 붙여넣기 | 공문, 보고서, 민원, 메일 본문이 길게 이어짐 | `blocked` 또는 `needs_security_review` |
| 실제 문서 제목 | 실제 기관명, 실제 사업명, 실제 문서명이 결합됨 | `needs_security_review` |
| 실제 수신ㆍ발신 정보 | 수신처, 발신처, 담당자, 연락처가 실제값처럼 보임 | `blocked` |
| 실제 문서번호 | 시행번호, 접수번호, 민원번호, 사건번호 의심값 | `blocked` |
| 실제 붙임명 | 실제 파일명, 실제 첨부 제목, 실제 내부 자료명 | `needs_security_review` |
| 실제 날짜ㆍ금액ㆍ수량 | 일정, 예산, 실적, 물량이 구체값으로 제시됨 | `needs_security_review` |
| 실제 내부 운영정보 | 내부 시스템, 계정, 권한, 장비, 차량, 배치 정보 | `blocked` |

실제 원문 의심 상태에서는 입력 내용을 그대로 로그에 복사하지 않습니다.

## 비식별 입력 확인 절차

입력은 다음 순서로 확인합니다.

1. 실제 원문 전체 또는 긴 발췌가 있는지 확인
2. 개인정보, 민원정보, 사건정보, 문서번호 의심값이 있는지 확인
3. 실제 기관명, 관서명, 업체명, 담당자명이 있는지 확인
4. 내부 운영정보, 계정, 권한, 시스템명, 차량번호, 장비 세부값이 있는지 확인
5. 예산, 일정, 실적, 수량이 실제값인지 placeholder인지 확인
6. 문서 유형이 허용 범위인지 확인
7. A/B/C/D 보안등급 후보를 보수적으로 판단
8. `ready_for_draft`, `needs_more_input`, `needs_security_review`, `blocked` 중 하나로 상태를 둠

## A/B/C/D 판단 절차

| 판단 단계 | 질문 | 판단 |
|---|---|---|
| 1 | 실제 개인정보 또는 고유식별정보가 있는가 | 있으면 D 후보 |
| 2 | 민원, 사건, 접수, 문서번호가 실제값처럼 보이는가 | 있으면 D 후보 |
| 3 | 내부 운영정보 또는 비공개 판단이 드러나는가 | 있으면 C 또는 D 후보 |
| 4 | 실제 기관, 업체, 담당자, 금액으로 특정 가능한가 | 있으면 B 이상 보수 검토 |
| 5 | placeholder 기반 구조만 남아 있는가 | A/B 후보 |
| 6 | 비식별 후에도 특정 상황을 추정할 수 있는가 | 있으면 C/D 후보 |

등급 판단이 애매하면 다음 원칙을 따릅니다.

- A/B가 애매하면 B
- B/C가 애매하면 C
- C/D가 애매하면 D

## 상태 결정 기준

| 상태 | 사용 조건 | 다음 동작 |
|---|---|---|
| `ready_for_draft` | 실제값 없음, placeholder 기반, A/B 후보, 필수 확인값 분리 | 로컬 초안 생성 가능 |
| `needs_more_input` | 실제값은 없지만 필수 정보가 부족함 | `[확인 필요]` 유지, 추가 입력 요청 |
| `needs_security_review` | 실제값 의심, B/C 경계, 내부정보 가능성 | 사람 보안 검토 전 외부 전송 금지 |
| `blocked` | 실제 원문, 개인정보, D등급 정보, 자동화 금지 요청 | 처리 중단, 비식별 재작성 요청 |

`needs_security_review`와 `blocked`는 외부 서비스 전송 후보에서 제외합니다.

## 진행 가능한 입력 기준

다음 조건을 모두 만족해야 외부 전송 전 검토 후보로 둘 수 있습니다.

- 실제 원문 없음
- 실제 개인정보 없음
- 실제 기관명, 관서명, 업체명, 담당자명 없음
- 실제 문서번호, 민원번호, 접수번호, 사건번호 없음
- 실제 날짜, 예산, 실적, 수량은 `[확인 필요]` 또는 placeholder로 분리
- 문서 유형은 허용된 `document_type`으로 표현
- 보안등급은 A/B 후보
- `human_review_required`는 `true`
- 자동 발송, 결재, 계약, 예산 집행, 법률 판단 요청 없음

## 보류 또는 차단 입력 기준

| 상황 | 상태 | 기준 |
|---|---|---|
| 실제 원문 전체 또는 일부 붙여넣기 | `blocked` | 외부 처리 금지 |
| 실제 개인정보 의심 | `blocked` | 비식별 재작성 필요 |
| 실제 문서번호 또는 사건번호 의심 | `blocked` | 외부 처리 금지 |
| 내부 운영정보 의심 | `blocked` | 외부 처리 금지 |
| 실제값인지 placeholder인지 불명확 | `needs_security_review` | 사람 보안 검토 필요 |
| B/C 경계 | `needs_security_review` | 외부 전송 전 보류 |
| C등급 후보 | `needs_security_review` | 내부 참고만 가능 |
| D등급 후보 | `blocked` | 외부 처리 금지 |

## 사용자 재작성 요청 기준

보류 또는 차단 시에는 실제 입력 내용을 반복하지 않고 다음 방향만 안내합니다.

허용 안내:

- 실제값을 제거하고 placeholder로 바꿔 다시 작성
- 일정, 예산, 실적, 담당자는 `[확인 필요]`로 분리
- 문서 유형만 `one_page_report`, `project_plan`, `result_report`, `review_report` 중 하나로 표시
- 실제 원문은 붙여넣지 않고 업무 목적과 필요한 항목만 일반화

금지 안내:

- 감지된 실제값을 다시 보여주기
- 원문 일부를 요약해 재출력하기
- 민감정보를 어떻게 우회하면 되는지 설명하기
- 실제 문서번호, 담당자명, 기관명을 예시로 다시 쓰기

## 외부 전송 전 사람 확인 항목

외부 전송은 현재 기본값 `전송하지 않음`입니다.

향후 별도 승인 후에도 외부 전송 후보로 검토하려면 사람이 최소한 다음을 확인해야 합니다.

| 확인 | 기준 |
|---|---|
| 실제 원문 없음 | 원문 전체, 발췌, 원문 요약이 없음 |
| 개인정보 없음 | 이름, 연락처, 이메일, 주소, 고유식별정보 없음 |
| 기관정보 없음 | 실제 기관명, 관서명, 부서명, 담당자명 없음 |
| 문서번호 없음 | 문서번호, 민원번호, 사건번호, 접수번호 없음 |
| 내부정보 없음 | 내부망, 계정, 권한, 장비, 차량, 운영정보 없음 |
| 보안등급 | A/B 후보만 검토 |
| 상태 | `ready_for_draft` 또는 허용된 `needs_more_input` |
| 로그 | 실제값 없이 상태와 범주만 기록 |
| preview | 사람이 초안과 placeholder 상태를 확인 |

하나라도 확인되지 않으면 외부 전송을 보류합니다.

## 다음 작업 후보

다음 단계는 실제 연동 구현이 아니라 외부 전송 없는 no-send dry-run 기준을 문서로 점검하는 것입니다.

검토 질문:

- 외부 전송 없이 어떤 상태값과 로그만 확인할지

- `전송하지 않음` 상태를 반복 운영 로그에 어떻게 표시할지
- 다음 단계에서도 코드 변경 없이 문서 기준만 보강할 수 있는지

# 비식별 문체 샘플

## 목적

실제 공문 원문을 사용하지 않고, 문체ㆍ구조ㆍ표현 방식만 학습할 수 있는 비식별 샘플을 정리합니다.

## 작성 원칙

- 실제 사건, 기관, 사람, 차량, 장비, 예산, 날짜를 넣지 않습니다.
- 구체 사실 대신 업무 구조를 나타내는 placeholder를 사용합니다.
- 원문을 변형한 문장이 아니라 새로 작성한 안전한 문체 샘플만 사용합니다.

## 공문 문체 샘플

```text
[업무 발생 배경]에 따라 [대상 기관]의 [점검 대상] 현황을 확인하고, 향후 관리 개선을 위한 기초자료로 활용하고자 합니다.
```

```text
이에 각 대상 기관에서는 아래 사항을 확인하여 [제출기한 확인 필요]까지 제출하여 주시기 바랍니다.
```

## 조사 요청 문체 샘플

```text
조사 항목 중 내부 기준 확인이 필요한 사항은 `[내부 기준에 따라 확인 필요]`로 표시하고, 외부 AI 처리용 초안에는 실제 내부 운영정보를 포함하지 않습니다.
```

## 업체 메일 문체 샘플

```text
본 요청은 사전 검토를 위한 자료 요청이며, 계약 또는 업체 선정이 확정된 사항은 아닙니다.
```

```text
회신 시 개인정보, 참석자 명단, 내부자료 또는 민감정보가 포함된 자료는 제외하여 주시기 바랍니다.
```

## 민원 관련 문체 샘플

```text
붙임 자료는 비식별 처리된 민원 요지 자료로 한정하며, 민원인 성명, 연락처, 주소, 민원번호, 접수번호 등 식별 가능 정보는 포함하지 않습니다.
```

## 사용 금지 샘플

다음 방식은 사용하지 않습니다.

- 실제 날짜를 포함한 사건 설명
- 실제 관서명 또는 업체명처럼 보이는 명칭
- 실제 수량 또는 예산처럼 보이는 값
- 특정 민원인을 식별할 수 있는 서술

# 사용자 안내 묶음 Closeout

## 목적

사용자 운영 A-to-Z 안내, quick start, 리허설 체크리스트, 비식별 요청 예시, 리허설 경계 문서를 하나의 push 가능한 문서 묶음으로 확정합니다.

이 문서는 실제 업무값 입력, 실제 HWPX 생성, 실제 외부 연동을 위한 문서가 아닙니다.

## 포함 문서

| 문서 | 역할 |
|---|---|
| `docs/140_user_operation_atoz_guide.md` | 사용자가 직접 할 일, Codex가 처리할 일, 보류할 일 정리 |
| `checklists/user_operation_atoz_rehearsal_checklist.md` | 실제 HWPX 없이 따라 하는 사용자 리허설 체크리스트 |
| `docs/141_user_rehearsal_prompt_examples.md` | 보고서 4종 비식별 요청 예시 |
| `docs/142_user_guidance_integrated_review.md` | 사용자 안내 3종 통합 점검 |
| `docs/143_user_quick_start.md` | 처음 볼 순서와 바로 멈출 지점 |
| `docs/144_quick_start_rehearsal_boundary.md` | 문서만으로 가능한 확인과 HWPX 열람 필요 지점 분리 |

## 확정한 기준

| 기준 | 판단 |
|---|---|
| `[사용자 확인 필요]` | 사람이 직접 판단하거나 한컴에서 확인해야 하는 항목으로 유지 |
| `[Codex 처리 가능]` | 비식별 placeholder 기반 구조화와 검토 목록 정리로 제한 |
| `[보류]` | 실제 원본, 실제 표 데이터, 외부 연동, 실제 발송ㆍ결재ㆍ집행 제외 |
| HWPX 열람 | 사용자가 직접 확인해야 하는 지점으로 분리 |
| Git 제외 | local template, output, normalizers output은 Git 제외 대상으로 유지 |

## 문서만으로 가능한 것

- 현재 PoC의 작동 범위 확인
- 처음 열 문서와 진행 순서 확인
- 보고서 4종 비식별 요청 예시 선택
- 실제값을 넣으면 안 되는 항목 확인
- `missing_fields`가 본문 실제값이 아니라 검토 목록임을 확인
- HWPX 열람이 필요한 지점과 아닌 지점 분리

## 사용자 직접 확인이 필요한 것

- 실제 기관 양식을 사용할지 여부
- 저장소 밖 비식별 작업 복사본 준비 여부
- 한컴에서 파일 열림, 글자 겹침, 줄바꿈, 표 폭, 여백 확인
- 기관 표준 글꼴, 자간, 줄간격, 문단 간격 값
- 표 내부 데이터 자동화를 Excel/한셀 후보로 따로 진행할지 여부

## 계속 보류하는 것

- 실제 HWPX/HWP 원본 저장소 추가
- 실제 HWPX output 재생성
- 실제 Excel/한셀 파일 생성
- 실제 표 데이터, 수량, 금액, 대상 목록 작성
- 실제 이메일 발송
- API, Make.com, Gmail, Outlook 실제 연동
- 실제 발송, 결재, 계약, 예산 집행, 법률 판단 자동화

## 검증 기준

이번 묶음은 다음 검증을 통과해야 push 단위로 봅니다.

- 문서 공백 오류 없음
- 민감정보 의심 패턴 없음
- output과 local HWPX 파일 Git 제외 유지
- 코드, renderer, normalizer, fixture 변경 없음
- 실제 HWPX output 또는 실제 HWPX 원본 추가 없음

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- Email, API, Make.com 연동 없음

## 판단

사용자 안내 묶음은 현재 단계에서 push 가능한 문서 산출물입니다.

다음 단계는 실제 HWPX 생성이 아니라, 사용자가 이 안내 묶음을 기준으로 실제 HWPX 없이 한 번 리허설할 수 있도록 유지하거나, 비식별 작업 복사본이 준비될 때 수동 preview로 넘어가는 것입니다.

# HWPX 일원화와 표 틀 우선 결정

## 목적

이 문서는 실제 양식 수동 리허설을 앞두고 HWPX 보고서 자동화 범위와 표 처리 범위를 다시 고정합니다.

이번 결정은 renderer, normalizer, fixture, routing, HWPX payload, output을 변경하기 위한 지시가 아닙니다. 실제 기관 HWPX/HWP 원본, 실제 문서 원문, 실제 개인정보도 저장소에 추가하지 않습니다.

## 결정 요약

| 항목 | 결정 |
|---|---|
| 최종 보고서 형식 | HWPX 일원화 유지 |
| DOCX/Word 변환 흐름 | 공식 흐름으로 채택하지 않음 |
| HWPX 표 | 현재 단계에서는 표 틀과 배치 검수 대상으로만 취급 |
| 표 내부 데이터 | 향후 Excel/한셀 연동 후보로 분리 |
| 현재 수동 preview | 표 위치, 폭, 줄바꿈, 겹침, 여백만 확인 |
| 실제 값 기록 | 금지 |

## HWPX 일원화 원칙

공공기관 실무 제출물과 보고서 양식의 중심 형식은 HWPX로 유지합니다.

DOCX/Word 저장 기능은 참고 가능하지만, 현재 프로젝트의 공식 흐름으로 채택하지 않습니다. DOCX를 중간 변환 형식으로 사용하면 일시적으로 편할 수 있으나, HWPX 최종 양식의 문단 배치, 표, 번호체계, 한컴 열람 결과를 다시 확인해야 하므로 현재 단계의 기준을 흐릴 수 있습니다.

따라서 현재 흐름은 다음처럼 유지합니다.

```text
비식별 입력
→ AI 초안 JSON
→ HWPX placeholder 템플릿
→ HWPX 초안 preview
→ 사람 검토
```

## 표 처리 범위

현재 단계에서 HWPX 보고서 안의 표는 자동 계산이나 세부 데이터 입력 대상이 아니라 문서 레이아웃의 틀로 봅니다.

확인할 항목:

- 표가 있어야 할 위치에 있는지
- 표 폭이 문서 폭 안에서 과도하게 좁거나 넓지 않은지
- 셀 안 짧은 문구가 겹치지 않는지
- 표 주변 문단과 여백이 어색하지 않은지
- 표가 본문 번호체계나 제목을 밀어내지 않는지

현재 단계에서 하지 않는 항목:

- 표 안 물품명, 수량, 금액, 대상 목록 자동 입력
- 실제 수량이나 예산 계산
- 실제 장비명 또는 내부 운영정보 반영
- 복잡한 표 셀별 placeholder 재구성
- 한컴 표를 Excel/한셀 데이터와 자동 동기화

## 표 내부 데이터 처리 원칙

표 내부에 실제 값이 많으면 모든 셀을 길게 placeholder화할 필요는 없습니다.

현재 단계에서는 보안 노출을 막을 정도로만 처리하고, 표의 형태를 확인할 수 있으면 충분합니다.

허용되는 처리:

- 실제 값 삭제
- 셀 일부 비움
- 실제값을 알 수 없는 짧은 중립 표기로 대체
- 레이아웃 확인용으로 `항목 A`, `항목 B`, `00`, `확인`, `[표 비식별]` 같은 짧은 표기 사용

금지되는 처리:

- 실제 물품명, 장비명, 기관명, 담당자명 기록
- 실제 수량, 금액, 일정, 실적 수치 기록
- 실제 표 내용을 요약하여 저장소 문서에 남김
- 실제 표 구조를 근거로 특정 기관이나 업무를 추정할 수 있게 작성
- 확인되지 않은 수량이나 금액을 임의 생성

## Excel/한셀 연동 후보

표 데이터는 향후 Excel 또는 한셀 연동 후보로 분리합니다.

역할 분리 기준:

| 계층 | 역할 |
|---|---|
| HWPX | 보고서 본문, 제목, 번호체계, 표 틀, 최종 제출 양식 |
| Excel/한셀 | 표 데이터 입력, 수량, 계산, 목록 관리 |
| AI 초안 | 문장 구조, 요약, 확인 필요 항목 표시 |
| 사람 검토 | 실제 값 입력 여부, 보안, 최종 승인 |

이 결정은 XLSX 조사표를 최우선 대상으로 올리는 의미가 아닙니다. 최우선 자동화 대상은 계속 HWPX 보고서 4종이며, 표 데이터 자동화만 별도 계층으로 보류합니다.

## 사용자 확인 범위

[사용자 확인 필요]

실제 양식 수동 preview에서 사용자가 확인할 항목은 아래로 제한합니다.

| 확인 | 항목 | 기록 방식 |
|---|---|---|
| [ ] | 표 위치가 문서 흐름 안에서 자연스러운가 | 문제 없으면 `table_scope: frame_only` |
| [ ] | 표 폭과 셀 폭이 크게 무너지지 않는가 | 실제 셀 값 기록 금지 |
| [ ] | 표 안 짧은 중립 문구가 겹치지 않는가 | 증상만 기록 |
| [ ] | 표가 본문 번호나 문단과 겹치지 않는가 | 항목 번호만 기록 |
| [ ] | 실제 표 값이 저장소 문서에 남지 않는가 | 실제 값이 있으면 기록 전 보류 |

문제가 없으면 다음처럼만 전달합니다.

```text
document_type: one_page_report
table_scope: frame_only
status: 문제 없음
```

문제가 있으면 실제 표 값 없이 증상만 전달합니다.

```text
document_type:
table_scope: frame_only
section:
gap_type: 표 폭
severity:
observed_symptom:
```

## 계속 보류하는 범위

- 실제 기관 HWPX/HWP 원본 Git 추가
- 실제 HWPX output 재생성
- 실제 원문, 실제 기관명, 실제 문서번호 기록
- 실제 표 데이터, 수량, 예산, 실적 수치 기록
- DOCX/Word 변환을 공식 흐름으로 채택
- renderer, normalizer, fixture, routing, HWPX payload 변경
- Email, API, Make.com 실제 연동 구현

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- 실제 파일명, 내부 경로, 원본 양식명 추가 없음
- API 키, 토큰, 인증정보 예시값 추가 없음
- Email, API, Make.com 연동 구현 없음

## 결론

현재 단계의 공식 방향은 HWPX 일원화입니다.

다만 HWPX 안의 표는 세부 데이터 자동화 대상이 아니라 표 틀과 배치 검수 대상으로 제한합니다. 표 내부 데이터 자동화는 향후 Excel/한셀 연동 후보로 분리하고, 지금은 저장소 밖 한컴 preview에서 표 틀만 실제값 없이 확인합니다.

# XLSX 조사표 템플릿 PoC 설계

## 목적
- AI가 생성한 현황조사표 JSON을 XLSX 조사표 템플릿에 삽입하기 위한 사전 설계 문서입니다.
- 이번 단계에서는 코드 구현이나 실제 XLSX 파일 생성을 하지 않습니다.
- 실제 기관 양식이나 실제 조사자료를 사용하지 않고 placeholder 기반으로 구조만 정의합니다.
- 향후 openpyxl 등 XLSX 처리 도구를 사용할 수 있지만, 이번 문서에는 코드 작성 없이 설계만 기록합니다.

## 기본 원칙
- 실제 원문 자료를 사용하지 않습니다.
- 실제 관서명, 차량번호, 장비명, 수량, 담당자명은 사용하지 않습니다.
- 확인되지 않은 값은 [확인 필요]로 표시합니다.
- 개인정보와 내부 운영정보는 XLSX 템플릿에 직접 입력하지 않습니다.
- XLSX는 조사표, 작성요령, 체크리스트 중심으로 설계합니다.
- AI 출력 JSON과 XLSX 셀 매핑을 분리합니다.
- 서식은 XLSX 템플릿이 담당하고, AI는 내용 구조만 제공합니다.
- 생성된 XLSX는 사람이 최종 검토해야 합니다.

## 대상 문서 유형
이번 PoC의 대상:
- 현황조사표
- 작성요령
- 제출 전 체크리스트

대상에서 제외할 것:
- 실제 관서별 운영현황
- 실제 차량번호
- 실제 장비명과 수량
- 실제 민원자료
- 실제 업체 견적자료
- 실제 개인정보가 포함된 신청서 또는 명단

## XLSX 파일 구성 초안

| 시트명 | 역할 | 비고 |
|---|---|---|
| 조사표 | 현황조사 항목 입력용 | placeholder 기반 |
| 작성요령 | 조사표 작성 방법 안내 | 개인정보ㆍ민감정보 제외 안내 포함 |
| 검수체크리스트 | 제출 전 확인 항목 | 민감정보 포함 여부 점검 |
| 메타정보 | 생성 정보 및 확인 필요 항목 | 실제 개인정보 없음 |

## 시트 1. 조사표 구조

상단 메타 영역:
- 문서 제목
- 조사 기준일
- 작성 단위
- 제출 기한
- 제출 방법
- 담당부서
- 공용 연락처
- 보안 유의사항

표 영역 열 예시:
- 연번
- 구분
- 항목 대분류
- 상태
- 보완 필요 여부
- 조치 필요사항
- 비고

주의:
실제 관서명, 차량번호, 장비명, 수량, 위치, 운영방식은 포함하지 않습니다.

## 시트 2. 작성요령 구조

다음 항목을 포함합니다.
- 작성 목적
- 작성 기준
- 항목별 작성 방법
- 상태 구분 기준
- 비고란 작성 기준
- 제외해야 할 정보
- 문의처 placeholder

제외해야 할 정보:
- 개인정보
- 차량번호
- 세부 장비명
- 장비별 수량
- 순찰구역
- 내부 시스템 정보
- 민원번호 또는 접수번호
- 업체 견적 금액

## 시트 3. 검수체크리스트 구조

다음 체크 항목을 포함합니다.
- 실제 관서명이 포함되지 않았는가
- 차량번호가 포함되지 않았는가
- 개인 이름과 연락처가 포함되지 않았는가
- 세부 장비명과 수량이 포함되지 않았는가
- 내부 운영정보가 포함되지 않았는가
- 민원번호 또는 접수번호가 포함되지 않았는가
- 첨부자료에 민감정보가 포함되지 않았는가
- 담당자 검토가 완료되었는가
- 내부 승인 절차가 필요한지 확인했는가

## 시트 4. 메타정보 구조

다음 항목을 포함합니다.
- request_id
- document_type
- generated_at_placeholder
- draft_status
- human_review_required
- risk_level
- missing_fields
- source_policy
- renderer_version_placeholder

generated_at_placeholder는 실제 날짜가 아니라 [생성일시 확인 필요]로 표시합니다.

## AI 출력 JSON과 XLSX 셀 매핑

docs/18_ai_output_json_schema.md의 현황조사표 JSON 스키마를 기준으로 XLSX 셀 매핑 초안을 작성합니다.

| JSON 필드 | XLSX 시트 | 셀 또는 영역 | 처리 방식 |
|---|---|---|---|
| title | 조사표 | 제목 영역 [확인 필요] | 텍스트 삽입 |
| standard_date | 조사표 | 조사 기준일 셀 [확인 필요] | [확인 필요] 허용 |
| columns | 조사표 | 표 머리글 행 [확인 필요] | 열 이름 생성 |
| rows | 조사표 | 표 본문 영역 [확인 필요] | 행 반복 생성 |
| privacy_notice | 작성요령 | 제외 정보 안내 영역 [확인 필요] | 텍스트 삽입 |
| checklist | 검수체크리스트 | 체크 항목 영역 [확인 필요] | 행 반복 생성 |
| missing_fields | 메타정보 | 확인 필요 항목 영역 [확인 필요] | 목록 삽입 |
| security_review.risk_level | 메타정보 | 보안 위험도 영역 [확인 필요] | low/medium/high/blocked |

실제 셀 주소는 아직 확정하지 않고 [확인 필요]로 표시합니다.

## XLSX PoC 테스트 시나리오

테스트 코드는 작성하지 않고 시나리오만 정의합니다.

테스트 1:
최소 행 3개짜리 조사표 JSON을 XLSX 템플릿에 넣을 수 있는지 확인합니다.

테스트 2:
rows가 0개일 때 빈 조사표가 깨지지 않는지 확인합니다.

테스트 3:
missing_fields가 여러 개일 때 메타정보 시트에 정상 표시되는지 확인합니다.

테스트 4:
security_review.risk_level이 high 또는 blocked일 때 렌더링을 중단하거나 경고 표시할 수 있는지 확인합니다.

테스트 5:
개인정보 또는 금지 필드가 rows에 포함될 경우 렌더링 전 차단할 수 있는지 확인합니다.

## XLSX 렌더링 전 차단 조건

다음 조건이 있으면 렌더링을 중단하거나 사람 검토로 보내야 합니다.

- contains_personal_info가 true
- contains_sensitive_info가 true
- risk_level이 high 또는 blocked
- rows에 실제 차량번호로 보이는 값이 포함됨
- rows에 실제 연락처로 보이는 값이 포함됨
- rows에 민원번호 또는 접수번호로 보이는 값이 포함됨
- 내부 시스템명, 계정, 비밀번호, API 키가 포함됨

주의:
실제 예시값은 만들지 않습니다.

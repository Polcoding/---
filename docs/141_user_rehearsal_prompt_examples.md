# 사용자 리허설 요청 예시

## 목적

`docs/140_user_operation_atoz_guide.md`와 `checklists/user_operation_atoz_rehearsal_checklist.md`를 따라 실제값 없이 Codex에 요청할 수 있는 비식별 예시를 정리합니다.

이 문서는 실제 업무 원문, 실제 개인정보, 실제 기관명, 실제 문서번호, 실제 예산액, 실제 일정, 실제 실적, 실제 표 데이터를 입력하기 위한 문서가 아닙니다.

## 공통 사용 원칙

- 실제 원문을 붙여넣지 않습니다.
- 실제 개인정보, 기관명, 부서명, 담당자명, 연락처, 이메일을 넣지 않습니다.
- 실제 문서번호, 접수번호, 민원번호, 사건번호를 넣지 않습니다.
- 실제 예산액, 계약 금액, 수량, 대상 목록을 넣지 않습니다.
- 확인되지 않은 값은 `[확인 필요]`로 둡니다.
- 표 데이터는 `[표 데이터 별도 확인 필요]` 또는 `해당 없음` 후보로만 표시합니다.
- HWPX 파일 열람이 필요한 지점은 `[사용자 확인 필요]`로 남깁니다.

## 공통 요청 머리말

아래 문구를 각 문서 유형 예시 앞에 붙여 사용할 수 있습니다.

```text
아래 입력은 비식별 placeholder 기반입니다.
실제 원문, 개인정보, 기관명, 문서번호, 실제 예산액, 실제 일정, 실제 실적, 실제 표 데이터는 포함하지 않았습니다.
확인되지 않은 값은 [확인 필요]로 유지해 주세요.
AI는 초안 구조화만 수행하고 실제 발송, 결재, 계약, 예산 집행, 법률 판단은 하지 않습니다.
```

```text
[비식별 업무] one_page_report 초안 구조를 정리해 주세요.

제목 힌트: [비식별 업무 원페이지 보고]
보고 목적: [확인 필요]
추진 배경: [비식별 배경 요약]
주요 내용:
1. [주요 내용 확인 필요]
2. [주요 내용 확인 필요]
3. [주요 내용 확인 필요]
검토 의견: [검토 의견 확인 필요]
문제점 및 고려사항: [고려사항 확인 필요]
향후 계획: [후속조치 확인 필요]
조치 필요사항: [조치 필요사항 확인 필요]
표 데이터 후보: [요약 표 필요 여부 확인 필요 또는 해당 없음]

[사용자 확인 필요]
- 보고 목적
- 주요 내용
- 향후 계획 또는 조치 필요사항
- 한컴 열람 시 글자 겹침, 항목 순서, 줄바꿈, 여백

[Codex 처리 가능]
- 개조식 원페이지 보고서 초안
- 확인 필요 항목 목록화
- 보안 주의문구 유지

[보류]
- 실제 기관 양식 원본 사용
- 실제 표 데이터 자동 작성
- 실제 발송 또는 결재
```

## review_report 예시

`review_report`는 보안 검토가 필요한 문서 유형입니다. 보안 검토 승인 신호가 없으면 HWPX 렌더링하지 않습니다.

```text
[비식별 사안] review_report 초안 구조를 정리해 주세요.

제목 힌트: [비식별 사안 검토보고]
검토 배경: [비식별 배경 요약]
검토 범위: [검토 범위 확인 필요]
검토 항목:
1. [검토 항목 확인 필요]
2. [검토 항목 확인 필요]
검토 의견: [검토 의견 확인 필요]
위험요소:
1. [위험요소 확인 필요]
2. [추가 검토 필요사항 확인 필요]
필요 검토: [법무ㆍ계약ㆍ개인정보 검토 필요 여부 확인]
표 데이터 후보: [검토항목 목록ㆍ위험요소 매트릭스 필요 여부 확인 필요 또는 해당 없음]
후속조치: [후속조치 확인 필요]

[사용자 확인 필요]
- 검토 범위
- 필요 검토 주체
- placeholder 렌더링 보안 승인 여부
- 한컴 열람 시 항목 순서, 줄바꿈, 글자 겹침

[Codex 처리 가능]
- 승인 조건이 있는 검토보고서 placeholder 초안
- 위험요소와 후속조치 개조식 정리
- 보안 검토 필요 상태 표시

[보류]
- 실제 검토의견 원문 처리
- 실제 승인자명, 결재번호, 문서번호 입력
- 보안 검토 없는 HWPX 렌더링
```

## 보안 검토 승인 신호 예시

아래 예시는 실제 승인 기록이 아닙니다. placeholder 렌더링 허용 여부를 나타내는 비식별 신호 후보입니다.

```json
{
  "security_gate": {
    "human_security_review_completed": true,
    "review_basis": "[비식별 보안 검토 완료]",
    "approved_for_placeholder_rendering": true
  }
}
```

이 신호에는 실제 승인자명, 실제 결재번호, 실제 검토의견 원문을 넣지 않습니다.

## 요청 후 확인 문구

Codex 결과를 받은 뒤에는 아래 기준으로 봅니다.

```text
[사용자 확인 필요]
- 확인 필요 항목이 실제값처럼 확정되지 않았는가
- 한컴 열람이 필요한 경우 글자 겹침, 항목 순서, 줄바꿈, 여백을 확인했는가
- GitHub Desktop Changes에 실제 HWPX 원본 또는 output이 보이지 않는가

[Codex 처리 가능]
- missing_fields 검토 목록 분리
- 안전 중단 상태 설명
- output과 local template Git 제외 여부 확인

[보류]
- 실제 표 데이터 작성
- 실제 외부 발송
- 실제 결재 또는 계약 판단
```

## 완료 기준

- 보고서 4종 요청 예시가 모두 placeholder 기반임
- 실제 개인정보, 기관정보, 문서번호, 원문, 표 데이터가 없음
- `[사용자 확인 필요]`, `[Codex 처리 가능]`, `[보류]` 구분 유지
- HWPX 열람이 필요한 지점은 사용자 확인으로 분리
- 실제 발송, 결재, 계약, 예산 집행, 법률 판단 자동화 없음

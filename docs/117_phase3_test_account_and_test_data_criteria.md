# Phase 3 테스트 계정과 테스트 데이터 기준

## 목적

Phase 3에서 실제 외부 연동을 구현하기 전에 테스트 계정과 테스트 데이터의 기준을 문서로 구체화합니다.

이 문서는 구현 지시서가 아닙니다. 실제 OpenAI API 호출, Make.com 시나리오, GmailㆍOutlookㆍSMTP 연동, 실제 이메일 자동 발송, 실제 계정 연결은 계속 금지합니다.

## 검토 결론

테스트 계정과 테스트 데이터 기준은 문서 기준으로 구체화할 필요가 있습니다.

현재 결론:

- 실제 업무 계정은 연결하지 않습니다.
- 테스트 계정도 실제 연결하거나 인증하지 않습니다.
- 테스트 데이터는 placeholder 기반으로만 정의합니다.
- 외부 서비스 전송은 기본값 `전송하지 않음`으로 유지합니다.
- 비용, 장애, 권한 기준이 확정되지 않으면 외부 연동 검토를 보류합니다.

## 계정 유형 구분

| 계정 유형 | 현재 기준 | 처리 |
|---|---|---|
| 실제 업무 계정 | 실제 발송ㆍ수신ㆍ업무 자료 접근 가능성이 있음 | 연결 금지 |
| 실제 기관 계정 | 실제 기관명, 실제 조직 도메인, 실제 업무 권한 포함 가능성 | 연결 금지 |
| 개인 실사용 계정 | 개인 메일, 연락처, 결제, 파일 접근 가능성 | 연결 금지 |
| 테스트 계정 후보 | 실제 업무 권한이 없는 별도 계정이어야 함 | 문서 기준 검토만 가능 |
| 샌드박스 또는 개발 계정 | 실제 업무 자료와 분리되어야 함 | 실제 연결 전 별도 승인 필요 |

현재 저장소에서는 어떤 계정도 실제로 연결하지 않습니다.

## 테스트 계정 최소 조건

향후 테스트 계정을 검토하더라도 다음 조건을 모두 만족해야 합니다.

| 조건 | 기준 |
|---|---|
| 업무 분리 | 실제 업무 메일, 실제 파일, 실제 연락처, 실제 일정에 접근하지 않음 |
| 권한 최소화 | 초안 생성 검토에 필요한 최소 권한만 부여 |
| 발송 차단 | 실제 이메일 발송, 자동 회신, 외부 공유 권한 없음 |
| 결제 분리 | 실제 업무 예산 또는 개인 결제수단과 분리 |
| 로그 분리 | 실제 원문, 개인정보, 실제 기관정보가 로그에 남지 않음 |
| 승인 | 사람 검토와 별도 승인 전 연결하지 않음 |
| 회수 가능 | 테스트 종료 후 권한 회수와 연결 해제가 가능해야 함 |

하나라도 확인되지 않으면 테스트 계정 연결은 보류합니다.

## 실제 업무 계정 연결 금지 기준

다음에 해당하면 테스트 목적이라도 연결하지 않습니다.

- 실제 수신함, 발신함, 주소록, 일정, 파일 저장소에 접근 가능
- 실제 업무 문서, 실제 원문, 실제 HWPX 원본에 접근 가능
- 실제 발송 권한, 공유 권한, 첨부 권한이 있음
- 실제 기관명, 실제 부서명, 실제 담당자명이 계정명이나 화면에 표시됨
- 실제 결제수단 또는 실제 예산 집행과 연결됨
- 계정 소유자, 권한 범위, 로그 보관 기준이 확인되지 않음

## 테스트 데이터 기준

테스트 데이터는 실제 업무 데이터를 줄인 것이 아니라, 처음부터 placeholder 기반으로 작성한 데이터여야 합니다.

허용:

- placeholder 기반 업무 지시
- `document_type` 같은 문서 유형 코드
- `ready_for_draft`, `needs_more_input` 같은 상태값
- `[확인 필요]`, `[사용자 확인 필요]` 같은 검토 표시
- 실제값이 없는 샘플 제목
- 실제 기관명, 문서번호, 개인정보가 없는 샘플 본문 구조

금지:

- 실제 공문 원문
- 실제 보고서 원문
- 실제 민원 내용
- 실제 개인정보
- 실제 기관명, 관서명, 업체명, 담당자명
- 실제 문서번호, 민원번호, 접수번호, 사건번호
- 실제 이메일 주소, 전화번호, 주소
- 실제 HWPX 원본 또는 output 파일
- 실제 API 요청ㆍ응답 전문
- 실제 첨부 파일명
- 실제 예산액, 견적 금액, 계약 조건

## 외부 전송 가능 범위

현재 단계의 기본값은 `외부 전송하지 않음`입니다.

향후 별도 승인 후에도 외부 전송 후보로 검토할 수 있는 것은 다음에 한정합니다.

| 항목 | 전송 후보 여부 | 기준 |
|---|---|---|
| placeholder 기반 짧은 테스트 문장 | 검토 가능 | 실제값 없음, 보안등급 A/B 후보 |
| document_type | 검토 가능 | 문서 유형 코드만 포함 |
| routing 상태값 | 검토 가능 | 본문 없이 상태값만 포함 |
| missing_fields 개수 | 검토 가능 | 실제 필드값 없이 개수만 포함 |
| 실제 원문 | 금지 | 외부 전송 금지 |
| 실제 HWPX 파일 | 금지 | 외부 전송 금지 |
| 실제 output 파일 | 금지 | 외부 전송 금지 |
| 실제 이메일 수신자 | 금지 | 외부 전송 금지 |
| API 키, 토큰, 인증정보 | 금지 | 저장소와 외부 입력 모두 금지 |

## 비용 기준

비용 기준은 아직 확정하지 않습니다.

보류 조건:

- 유료 요금제 필요 여부가 확인되지 않음
- 비용 한도와 사용량 확인 방식이 없음
- 실제 업무 예산 또는 개인 결제수단이 필요함
- 사용량 초과 시 중단 기준이 없음
- 비용 로그에 실제 계정정보가 남을 가능성이 있음

위 항목 중 하나라도 해당하면 실제 연동 검토를 보류합니다.

## 장애 기준

외부 서비스 장애 또는 불안정 상태에서는 자동 재시도나 우회 처리를 설계하지 않습니다.

보류 조건:

- 외부 서비스 장애 발생
- 응답 지연 또는 실패 반복
- rate limit 또는 quota 초과
- 요청 결과가 누락되거나 중복됨
- 장애 로그에 실제 입력이 포함될 가능성
- 실패 시 사람이 확인할 절차가 없음

장애 상태에서는 초안 생성 자동화를 진행하지 않고 사람 검토로 전환합니다.

## 권한 기준

권한 기준이 모호하면 더 보수적으로 처리합니다.

보류 조건:

- 계정 소유자 불명확
- 연결 앱 권한 범위 불명확
- 읽기 권한과 쓰기 권한 분리 불가
- 발송, 공유, 첨부, 삭제 권한이 포함됨
- 권한 회수 절차가 없음
- 로그 접근 권한이 정리되지 않음

실제 업무 계정 연결은 별도 승인 전까지 계속 금지합니다.

## 테스트 계정 검토 로그 항목

테스트 계정을 문서 기준으로 검토할 때도 실제 계정명은 기록하지 않습니다.

| 항목 | 허용 형식 |
|---|---|
| 계정 유형 | `테스트 계정 후보` / `샌드박스 후보` / `[확인 필요]` |
| 업무 계정 분리 여부 | `분리됨` / `분리 안 됨` / `[확인 필요]` |
| 발송 권한 여부 | `없음` / `있음` / `[확인 필요]` |
| 외부 전송 여부 | `전송하지 않음` / `[확인 필요]` |
| 비용 영향 | `없음` / `있음` / `[확인 필요]` |
| 보류 사유 | 범주만 기록 |

실제 계정명, 실제 이메일 주소, 실제 조직명, 실제 결제정보는 기록하지 않습니다.

## 코드 변경 판단

이번 기준 정리는 문서 기준으로 충분합니다.

변경하지 않는 항목:

- `normalizers/`
- `renderers/`
- `examples/json/`
- routing fixture
- HWPX payload mapper
- HWPX renderer output
- local HWPX template
- API, Make.com, Email 연동 코드

## 다음 작업 후보

다음 단계는 실제 연동 구현이 아니라 사용자 preview와 사람 승인 지점을 외부 전송 전 절차로 통합 점검하는 것입니다.

검토 질문:

- HWPX preview, 외부 전송 전 preview, Email 초안 preview의 사람 확인 지점을 어떻게 구분할지
- 승인 상태가 실제 결재나 발송 승인이 아님을 어떻게 반복 표시할지
- 외부 전송 전 최종 보류 조건을 어디에 기록할지
- 다음 단계에서도 코드 변경 없이 문서 기준만 보강할 수 있는지

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX 원본 파일 추가 없음
- 실제 계정명, 이메일 주소, 결제정보 추가 없음
- API 키, 토큰, 인증정보 예시값 추가 없음
- Email, API, Make.com 연동 구현 없음

## 결론

테스트 계정과 테스트 데이터 기준은 문서 기준으로 구체화했습니다.

현재 단계에서는 계정을 실제로 연결하지 않고, placeholder 기반 테스트 데이터와 실제 업무 데이터의 경계를 정리하는 수준에서 멈춥니다.

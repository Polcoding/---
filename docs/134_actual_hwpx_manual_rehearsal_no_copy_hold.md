# 실제 양식 수동 리허설 복사본 없음 보류 기록

## 목적

이 문서는 실제 양식 수동 리허설 단계에서 비식별 HWPX 작업 복사본이 아직 없음을 기록하고, 실제 한컴 preview 결과 기록을 보류하는 기준을 정리합니다.

이번 기록은 실제 기관 HWPX/HWP 원본, 실제 문서 원문, 실제 개인정보, 실제 파일명, 실제 내부 경로를 저장소에 남기기 위한 문서가 아닙니다.

## 확인 근거

사용자는 실제 양식 수동 리허설에 사용할 비식별 HWPX 작업 복사본이 없다고 명시했습니다.

사용자 확인 표현:

```text
복사본 없는데
```

## 현재 상태

| 항목 | 상태 |
|---|---|
| 실제 원본 | 저장소에 추가하지 않음 |
| 비식별 HWPX 작업 복사본 | 없음 |
| 한컴 preview 대상 파일 | 없음 |
| gap log 결과 | 기록 불가 |
| HWPX output 재생성 | 하지 않음 |
| renderer 또는 normalizer 변경 | 하지 않음 |

Codex가 확인할 수 있는 범위는 저장소 내부 상태와 Git 제외 규칙입니다. 저장소 밖 실제 원본이나 작업 복사본의 존재 여부는 사용자의 확인에 의존합니다.

## 판단

비식별 HWPX 작업 복사본이 없으므로 실제 양식 수동 preview를 진행하지 않습니다.

따라서 다음 항목은 기록하지 않습니다.

- `문제 없음` preview 결과
- 실제 양식 gap log
- 표 위치, 폭, 줄바꿈, 겹침에 대한 실제 한컴 확인 결과
- 실제 파일명 또는 저장소 밖 경로

복사본이 없는 상태에서 임의로 정상 통과 기록을 만들지 않습니다.

## 계속 유지하는 기준

- HWPX 보고서 4종은 계속 최우선 자동화 대상입니다.
- HWPX는 최종 보고서 형식으로 유지합니다.
- 표는 현재 단계에서 `table_scope: frame_only` 기준의 틀 검수 대상으로만 봅니다.
- 표 내부 데이터, 수량, 금액, 대상 목록 자동화는 향후 Excel/한셀 연동 후보로 분리합니다.
- 실제 원본, 실제 기관정보, 실제 문서번호, 실제 개인정보는 저장소에 남기지 않습니다.

## 복사본 없이 가능한 다음 내부 작업

실제 HWPX 작업 복사본이 없어도 다음 문서 작업은 진행할 수 있습니다.

| 작업 | 설명 |
|---|---|
| HWPX 본문과 표 데이터 역할 분리 | HWPX는 보고서 틀, Excel/한셀은 표 데이터 후보로 구분 |
| 표 데이터 입력 범위 정리 | 실제 값 없이 어떤 종류의 표 데이터가 향후 분리 대상인지 분류 |
| 사용자 입력 템플릿 보강 | 수량, 금액, 대상 목록은 임의 생성 금지와 확인 필요 기준 유지 |
| 보류 조건 정리 | 실제 원본 또는 복사본 없을 때 preview를 진행하지 않는 기준 유지 |

이 작업은 실제 Excel/한셀 파일을 만들거나 연동 코드를 작성하는 단계가 아닙니다.

## 나중에 복사본이 준비되면

사용자가 나중에 저장소 밖에서 비식별 HWPX 작업 복사본을 준비하면 다음 기준으로 재개합니다.

```text
document_type: one_page_report
table_scope: frame_only
status: 문제 없음
```

또는 문제가 있으면 실제 값 없이 증상만 기록합니다.

```text
document_type:
table_scope: frame_only
section:
gap_type:
severity:
observed_symptom:
```

## 계속 보류하는 범위

- 실제 기관 HWPX/HWP 원본 Git 추가
- 실제 HWPX output 재생성
- 실제 원문, 실제 기관명, 실제 문서번호 기록
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록 기록
- HWPX 표 내부 실제 데이터 자동 입력
- Excel/한셀 자동 연동 구현
- renderer, normalizer, fixture, routing, HWPX payload 변경
- Email, API, Make.com 실제 연동 구현

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- 실제 파일명, 내부 경로, 원본 양식명 추가 없음
- API 키, 토큰, 인증정보 예시값 추가 없음
- Email, API, Make.com 연동 구현 없음

## 결론

비식별 HWPX 작업 복사본이 없으므로 실제 양식 수동 리허설은 보류합니다.

다음 작업은 실제 파일 preview가 아니라, 저장소 내부에서 HWPX 보고서 본문과 Excel/한셀 표 데이터 후보의 역할 분리를 실제값 없이 문서화하는 것입니다.

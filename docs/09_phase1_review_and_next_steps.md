# 1단계 검수 및 다음 스텝

## 목적

Codex가 만든 1단계 산출물을 사용자가 검수하고, Custom GPT 테스트 결과를 바탕으로 다음 개선 작업을 진행하는 방법을 정리합니다.

## 1. Codex 1차 산출물 생성 후 사용자가 해야 할 일

1. 파일 목록을 확인합니다.
2. `AGENTS.md`의 작업 규칙을 확인합니다.
3. `README.md`에서 프로젝트 목적과 1단계 범위를 확인합니다.
4. 보안 키워드를 검색하여 실제 개인정보나 내부자료처럼 보이는 내용이 없는지 확인합니다.
5. `prompts/custom_gpt_instructions_v0.1.md`가 그대로 복사 가능한지 확인합니다.
6. `prompts/conversation_starters.md`의 대화 시작 문구를 확인합니다.
7. Codex에 2차 자체 점검을 요청합니다.

## 2. Custom GPT 생성 후 해야 할 일

1. Instructions를 복사합니다.
2. Conversation Starters를 복사합니다.
3. Knowledge는 비워 둡니다.
4. Capabilities는 초기 테스트에서 비활성화합니다.
5. Actions/Apps는 사용하지 않습니다.
6. Preview에서 테스트 1~5를 실행합니다.
7. GPT는 비공개로 저장합니다.

## 3. 테스트 후 해야 할 일

1. 테스트 결과를 `docs/05_evaluation_sheet.md` 기준으로 채점합니다.
2. 실패한 항목을 기록합니다.
3. GPT 출력에서 위험한 부분을 표시합니다.
4. `prompts/custom_gpt_instructions_v0.1.md` 개선 방향을 정리합니다.
5. 수정 후 같은 테스트를 다시 실행합니다.

## 4. 수정 요청 프롬프트 예시

아래 문구를 사용해 테스트 결과를 Codex에 다시 전달할 수 있습니다.

```text
아래는 Custom GPT 테스트 결과입니다.
평가표 기준으로 문제점을 분석하고, prompts/custom_gpt_instructions_v0.1.md를 개선해 주세요.

수정 목표:
1. 없는 정보는 더 명확히 [확인 필요] 처리
2. 위험 요청은 더 강하게 제한
3. 업체 메일에서 계약 확정처럼 보이는 표현 제거
4. 공문 문체를 더 개조식으로 정리
5. 짧은 지시에도 현재 정보만으로 초안 작성

테스트 입력:
[여기에 입력 붙여넣기]

GPT 출력:
[여기에 출력 붙여넣기]

평가 결과:
[여기에 점수와 의견 붙여넣기]
```

## 5. 다음 단계로 넘어가는 조건

- 테스트 5개 모두 통과
- 위험 요청 처리 성공
- 개인정보 비식별화 성공
- 허위 생성 없음
- 업체 선정ㆍ계약 확정 표현 없음
- 사람 최종 승인 구조 유지
- Knowledge 업로드 전 비식별 샘플 기준 확정
- 조직 보안정책 검토 완료

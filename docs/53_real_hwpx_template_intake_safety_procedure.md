# 실제 HWPX 양식 투입 전 안전 절차

## 목적

이 문서는 실제 기관 HWP/HWPX 양식을 참고해야 하는 단계에 들어가기 전에, 원본 노출 없이 placeholder 템플릿을 준비하기 위한 안전 절차를 정리합니다.

현재 원칙은 실제 양식 원본을 Git에 추가하지 않고, 실제 원문이나 개인정보를 외부 AI에 입력하지 않는 것입니다.

## 기본 원칙

최신 저장소 밖 실제 양식 후보 수동 절차와 보류 조건은 `docs/128_external_hwpx_candidate_manual_procedure_and_hold_criteria.md`를 함께 따릅니다.

- 실제 기관 HWP/HWPX 양식 원본은 저장소에 추가하지 않습니다.
- 실제 공문 원문, 보고서 원문, 민원자료, 내부 운영자료를 템플릿에 넣지 않습니다.
- 실제 기관명, 문서번호, 결재선, 직인, 로고, 담당자명, 연락처를 템플릿에 넣지 않습니다.
- 로컬 HWPX 템플릿은 `templates/hwpx/` 아래에서만 검증하며 Git 제외 상태를 확인합니다.
- output 산출물은 `renderers/hwpx_renderer/output/` 아래에서만 생성하고 Git 제외 상태를 확인합니다.
- 실제 서식값이 확인되지 않았으면 `[확인 필요]`로 둡니다.
- 판단이 애매하면 더 보수적으로 분류합니다.

## 투입 가능성 판단

실제 양식을 바로 쓰지 않고, 먼저 보안등급을 판단합니다.

| 등급 | 처리 원칙 |
|---|---|
| A | 공개 가능 자료만 참고 가능 |
| B | 비식별 처리 후 placeholder 템플릿 후보로 검토 가능 |
| C | 내부 참고만 가능, 외부 AI 입력 및 샘플화 금지 |
| D | 외부 처리 금지, 템플릿 작업 중단 |

B/C 또는 C/D 판단이 애매하면 더 높은 위험 등급으로 분류합니다.

## 작업 절차

1. 원본 분류
   - 실제 양식 원본인지 확인합니다.
   - 개인정보, 민원정보, 내부 운영정보, 대외비 표시 여부를 확인합니다.
   - C/D등급이면 작업을 중단합니다.

2. 로컬 복사본 생성
   - 원본은 수정하지 않습니다.
   - 작업용 복사본은 로컬에서만 다룹니다.
   - 복사본을 Git 추적 대상에 추가하지 않습니다.

3. 실제 내용 제거
   - 본문 원문을 모두 삭제합니다.
   - 실제 문서번호, 시행일자, 수신처, 담당자, 연락처를 삭제합니다.
   - 실제 기관명, 관서명, 업체명, 개인명, 장소명, 사건 맥락을 삭제합니다.
   - 직인, 로고, 결재선 등 식별 가능한 양식 요소는 제거하거나 placeholder로 대체합니다.

4. placeholder 삽입
   - 동적 값은 `{{title}}`, `{{background}}`, `{{main_points}}`처럼 중립적인 placeholder로 바꿉니다.
   - 실제 값을 유추할 수 있는 placeholder 이름은 사용하지 않습니다.
   - 여러 줄이 들어갈 placeholder는 가능한 한 서로 다른 문단에 배치합니다.

5. Git 제외 확인
   - GitHub Desktop Changes에 `.hwpx`, `.hwp`, output 파일이 나타나지 않아야 합니다.
   - 나타나면 즉시 작업을 중단하고 `.gitignore` 상태를 확인합니다.
   - 현재 저장소는 `templates/hwpx/.gitignore`와 `renderers/hwpx_renderer/output/.gitignore`로 로컬 HWPX와 output을 제외합니다.

6. 렌더러 검증
   - placeholder 기반 샘플 JSON으로만 렌더링합니다.
   - `remaining_placeholders`가 남는지 확인합니다.
   - 템플릿이 없으면 `template_required`로 안전 중단하는 상태를 허용합니다.

7. 한컴 수동 검수
   - output HWPX를 열어 문단 분리, 글자 겹침, 줄간격, 번호체계, 붙임 표기를 확인합니다.
   - 겹침이 있으면 실제 데이터가 아니라 placeholder 배치와 렌더러 변환 범위만 조정합니다.

8. 결과 문서화
   - 사용한 샘플 JSON, 문서 유형, 렌더링 상태, 수동 검수 결과를 문서화합니다.
   - 실제 원본 내용이나 실제 양식 이미지는 문서에 첨부하지 않습니다.

## 중단 조건

다음 중 하나라도 해당하면 작업을 중단합니다.

- 실제 개인정보가 포함되어 있음
- 실제 민원번호, 접수번호, 사건번호가 포함되어 있음
- 실제 문서번호, 시행일자, 담당자명, 연락처가 포함되어 있음
- 실제 기관명, 관서명, 업체명, 직인, 로고가 식별 가능하게 포함되어 있음
- 감사, 징계, 수사, 보안, 대외비, 비공개 문서임
- 실제 차량번호, 장비현황, 순찰구역, 내부 운영정보가 포함되어 있음
- GitHub Desktop Changes에 실제 HWP/HWPX 또는 output 파일이 표시됨
- 사용자가 실제 양식 원본을 Git에 추가하라고 명시하지 않았음

## 산출물 원칙

- Git에 남기는 것은 절차, 체크리스트, placeholder 설계 문서입니다.
- 실제 HWPX 템플릿과 output HWPX는 로컬에만 둡니다.
- 실제 양식 원본은 저장소 밖에서만 관리합니다.
- 향후 운영 단계에서도 AI가 실제 발송, 결재, 계약, 업체 선정, 예산 집행, 법률 판단을 수행하지 않습니다.

# 수동 Preview 재개 게이트

## 목적

문서 기반 hold 상태에서 실제 HWPX 수동 preview로 넘어가도 되는지 판단하는 게이트를 정리합니다.

이 문서는 실제 업무용 HWPX 파일 생성 지시가 아닙니다. 실제 원문, 개인정보, 기관명, 문서번호, 실제 표 데이터, 실제 HWPX/HWP 원본을 기록하지 않습니다.

## 기본 판단

현재 기본값은 `preview 보류`입니다.

아래 조건이 모두 확인될 때만 `수동 preview 준비 가능`으로 바꿉니다.

| 조건 | 기본 상태 |
|---|---|

## 제공 샘플 PoC 확인 상태

사용자가 제공한 placeholder 없는 샘플 HWPX 2종은 저장소 밖 샘플 파일을 기준으로 profile-aware autofill PoC를 실행하고, ignored output을 한컴에서 수동 확인했습니다.

| 항목 | 상태 |
|---|---|
| 샘플양식1 목차 중복 anchor 회피 | 확인 완료 |
| 샘플양식1 글자 겹침 여부 | 사용자 확인 완료 |
| 샘플양식2 `❍ -` bullet 중복 제거 | 확인 완료 |
| 샘플양식2 bullet 표기 일관성 | 사용자 확인 완료 |
| output HWPX Git 제외 | 확인 완료 |

이 확인은 제공 샘플 HWPX에 대한 로컬 PoC 검수입니다. 실제 기관 양식, 실제 원본, 실제 개인정보가 포함된 파일에 대한 preview 재개 승인으로 보지 않습니다.

## 재개 가능 조건

수동 preview를 재개하려면 다음 조건이 모두 필요합니다.

1. 사용자가 저장소 밖 비식별 작업 복사본을 준비했다고 명시
2. 실제 원문, 개인정보, 기관명, 문서번호, 실제 표 데이터가 제거됨
3. 실제 HWPX/HWP 원본이 GitHub Desktop Changes에 보이지 않음
4. 로컬 HWPX 템플릿과 output 파일이 Git 제외 상태임
5. 사용자가 한컴에서 열람 검수를 진행할 수 있음
6. 검수 범위가 글자 겹침, 줄바꿈, 표 폭, 여백, placeholder 잔여 확인으로 제한됨
7. 외부 HWPX 자동 채우기 자료는 구조 분석 참고용으로만 두고 실제 원본 처리나 운영 의존성 도입으로 사용하지 않음

## 재개 불가 조건

아래 중 하나라도 해당하면 preview는 계속 보류합니다.

- 비식별 작업 복사본 준비 여부가 확인되지 않음
- 실제 원문 또는 개인정보 포함 가능성이 있음
- 실제 기관명, 문서번호, 담당자명, 수신자명 포함 가능성이 있음
- 실제 표 데이터, 수량, 금액, 대상 목록이 포함될 가능성이 있음
- 실제 HWPX/HWP 원본이 Changes에 보임
- 사용자가 한컴 열람을 진행할 수 없음
- Email/API/Make.com 등 외부 연동 요구가 섞임
- 외부 skill, MCP, `hwpx-cli`를 통해 실제 원문 또는 실제 기관 양식을 바로 처리하려는 요구가 섞임

## 사용자 확인 항목

`[사용자 확인 필요]`

- 저장소 밖 비식별 작업 복사본 준비 여부
- 실제값 제거 여부
- GitHub Desktop Changes 확인
- 한컴 파일 열람 가능 여부
- 글자 겹침, 줄바꿈, 표 폭, 여백, placeholder 잔여 확인

`[Codex 처리 가능]`

- 재개 조건 문서 기준 확인
- Git 제외 상태 확인 명령 실행
- 민감정보 의심 패턴 검색
- 검수 결과 기록 양식 준비
- 외부 HWPX 자동 채우기 자료가 구조 분석 참고용으로만 연결되는지 확인
- 비식별 HWPX 후보에 대해 `hwpx_template_structure_analyzer_poc.py --template ... --no-output` 구조 선확인

## 구조 선확인 절차

사용자가 저장소 밖 비식별 HWPX 작업 복사본을 준비했다고 명시한 경우에도, 실제 preview나 output 생성보다 먼저 구조 선확인을 수행합니다.

```powershell
python .\normalizers\hwpx_template_structure_analyzer_poc.py --template "[비식별 HWPX 후보 경로]" --no-output
```

확인 범위는 콘솔에 표시되는 구조 메타정보로 제한합니다.

- `scope`가 `structure_only_no_text_extraction`인지 확인
- `summary_output_skipped_no_output_flag`가 표시되는지 확인
- section, paragraph, table, placeholder 개수만 확인
- 본문 텍스트, 실제 원문, 개인정보, 실제 표 데이터는 기록하지 않음
- 구조 선확인 결과만으로 `수동 preview 준비 가능`으로 바꾸지 않음

구조 선확인에서 실제값 포함 가능성, Git 추적 대상 HWPX/output, 외부 도구를 통한 실제 원문 처리 요구가 보이면 preview는 계속 보류합니다.

`[보류]`

- 실제 원본 파일 Git 추가
- 실제 업무용 HWPX output Git 추가
- 실제 표 데이터 자동 작성
- 실제 외부 발송 또는 연동
- 외부 skill 또는 `hwpx-cli` 기반 실제 원본 자동 처리

## 재개 후 기록 원칙

수동 preview를 재개하더라도 기록은 다음 범위로 제한합니다.

- 파일명은 비식별 후보명 또는 placeholder명으로 기록
- 증상은 항목 번호와 현상만 기록
- 실제 문서 내용, 실제 표 데이터, 실제 개인정보는 기록하지 않음
- 글자 겹침, 줄바꿈, 여백, 표 폭 같은 레이아웃 현상만 기록

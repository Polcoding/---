# HWPX 보고서와 Excel/한셀 표 데이터 역할 분리 범위

## 목적

이 문서는 HWPX 보고서 자동화와 향후 Excel/한셀 표 데이터 처리를 어떤 역할로 나눌지 정리합니다.

실제 Excel/한셀 파일을 만들거나 자동 연동을 구현하는 문서가 아닙니다. 실제 기관 HWPX/HWP 원본, 실제 표 데이터, 실제 물품명, 실제 수량, 실제 금액, 실제 대상 목록도 저장소에 추가하지 않습니다.

## 기본 결정

| 구분 | 역할 |
|---|---|
| HWPX | 최종 보고서 형식, 제목, 본문, 번호체계, 표 틀, 검토 문구 |
| Excel/한셀 | 향후 표 데이터 입력, 수량, 금액, 대상 목록, 계산 후보 |
| AI 초안 | 비식별 지시 기반 문장 구조, 요약, 확인 필요 항목 표시 |
| 사람 검토 | 실제 값 입력 여부, 보안 검토, 최종 승인 |

HWPX 보고서가 프로젝트의 중심입니다. Excel/한셀은 HWPX 보고서 안의 표 내부 데이터를 별도 계층으로 관리할 필요가 생길 때 검토할 보조 후보입니다.

## HWPX가 담당할 범위

HWPX는 보고서의 읽히는 구조와 최종 양식을 담당합니다.

포함 범위:

- 문서 제목
- 보고 개요
- 추진 배경
- 주요 내용
- 검토 의견
- 문제점 및 고려사항
- 향후 계획
- 조치 필요사항
- 확인 필요 항목
- 보안 검토 표시
- 표가 들어갈 위치와 표 틀

HWPX에서 하지 않는 범위:

- 표 내부 실제 수량 자동 입력
- 실제 금액 계산
- 실제 대상 목록 관리
- 실제 물품명 또는 장비명 반영
- Excel/한셀 데이터 자동 동기화

## Excel/한셀 후보 범위

Excel/한셀은 향후 표 데이터가 반복적으로 필요할 때 검토할 수 있는 보조 계층입니다.

후보 범위:

| 후보 | 설명 | 현재 처리 |
|---|---|---|
| 수량 목록 | 물품, 자료, 대상별 수량 | 실제값 기록 금지 |
| 금액ㆍ예산 표 | 단가, 금액, 합계 | `[확인 필요]` 유지 |
| 일정 표 | 세부 일정, 날짜, 담당 | 실제 일정 입력 금지 |
| 대상 목록 | 기관, 부서, 대상자, 구역 | 실제 대상 기록 금지 |
| 결과 집계 | 실적, 참여 인원, 처리 결과 | 실제 실적 기록 금지 |
| 검토 매트릭스 | 위험요소, 검토항목, 조치상태 | 확정 판단 금지 |

현재 단계에서는 위 항목을 실제 데이터로 만들지 않습니다. 분리 후보로만 둡니다.

## 문서유형별 분리 기준

| 문서유형 | HWPX 본문 중심 | Excel/한셀 후보 |
|---|---|---|
| `one_page_report` | 보고 개요, 주요 내용, 검토 의견, 향후 계획 | 간단한 조치 목록, 요약 표 후보 |
| `project_plan` | 추진 배경, 세부 추진계획, 기대효과 | 일정표, 예산표, 대상 목록 후보 |
| `result_report` | 추진 결과, 주요 성과, 개선사항 | 실적 집계, 예산 집행 표 후보 |
| `review_report` | 검토 배경, 검토 의견, 보안 검토 조건 | 위험요소 매트릭스, 검토항목 목록 후보 |

후보라는 표현은 실제 자동화를 승인했다는 의미가 아닙니다.

## 사용자 입력 원칙

사용자가 실제 값을 외부 AI나 저장소에 그대로 넣지 않도록 입력 단계를 분리합니다.

허용되는 표현:

- `표 데이터는 별도 확인 필요`
- `수량은 [확인 필요]`
- `예산은 [확인 필요]`
- `대상 목록은 내부 검토 후 입력`
- `한셀 연동 후보`

금지되는 표현:

- 실제 물품명
- 실제 수량
- 실제 금액
- 실제 대상자 또는 담당자
- 실제 기관명 또는 부서명
- 실제 장비명 또는 차량번호
- 실제 내부 운영정보

## 향후 데이터 흐름 후보

향후 별도 승인 후 검토할 수 있는 흐름은 아래와 같습니다.

```text
비식별 업무 지시
→ HWPX 보고서 본문 초안
→ 표 데이터 필요 여부 표시
→ Excel/한셀 입력 후보 별도 관리
→ 사람 검토 후 실제 값 입력
→ HWPX 보고서 최종 검토
```

현재 단계에서는 이 흐름을 구현하지 않습니다.

## 안전 중단 기준

다음 중 하나라도 있으면 표 데이터 자동화 검토를 중단합니다.

| 중단 조건 | 처리 |
|---|---|
| 실제 표 데이터가 저장소에 들어가려 함 | 즉시 중단 |
| 실제 기관명, 담당자명, 대상 목록이 필요함 | 비식별 전까지 보류 |
| 수량, 금액, 일정이 확인되지 않음 | `[확인 필요]` 유지 |
| Excel/한셀 파일 생성이 필요해짐 | 별도 승인 전 보류 |
| HWPX renderer 변경이 필요해짐 | 별도 승인 전 보류 |
| 외부 API나 자동 발송과 연결하려 함 | 보안 검토 전 보류 |

## 다음에 정리할 수 있는 항목

복사본 없이 다음 단계에서 정리할 수 있는 항목은 아래와 같습니다.

- HWPX 보고서 4종 사용자 입력 템플릿에서 표 데이터 후보 표시
- `missing_fields`에 표 데이터 확인 필요 항목을 어떻게 보여줄지 검토
- Excel/한셀 후보가 필요한 경우와 필요 없는 경우 구분
- 실제값 없이 표 데이터 검토 checklist 작성

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- 실제 Excel/한셀 파일 추가 없음
- API 키, 토큰, 인증정보 예시값 추가 없음
- Email, API, Make.com 연동 구현 없음

## 결론

HWPX는 보고서 본문과 최종 양식을 담당하고, Excel/한셀은 향후 표 내부 데이터가 반복적으로 필요할 때 검토할 보조 계층으로 둡니다.

현재 단계에서는 실제 표 데이터 자동화나 파일 연동을 구현하지 않고, 역할 분리와 보류 기준만 문서화합니다.

# 문서 보안등급 분류 기준

## 목적

이 문서는 문서 패턴 추출, 비식별 샘플 작성, 내부 참고 여부를 판단하기 위한 A/B/C/D 보안등급 분류 기준을 정의합니다.

문서 등급은 실제 원문 내용의 민감도, 개인정보 포함 가능성, 내부 운영정보 포함 가능성, 외부 AI 처리 적합성을 기준으로 판단합니다.

판단이 애매한 경우에는 항상 더 보수적인 등급으로 분류합니다.

## 등급 요약

| 등급 | 의미 | 활용 방식 |
|---|---|---|
| A등급 | 공개 가능 수준 | 비식별 샘플 후보로 검토 가능 |
| B등급 | 비식별 후 제한적 활용 가능 | placeholder 치환 후 문체ㆍ구조만 샘플 후보로 검토 가능 |
| C등급 | 내부 참고만 가능 | 외부 AI 처리 및 샘플화 금지, 내부 검토용 참고만 가능 |
| D등급 | 외부 처리 금지 | 패턴 추출, 샘플화, 외부 AI 입력 모두 금지 |

## A등급: 공개 가능

### 기준

- 공개되어도 문제가 없는 일반 문서 구조 또는 서식
- 개인정보, 민감정보, 내부 운영정보, 비공개 판단 정보가 포함되지 않은 문서
- 기관ㆍ개인ㆍ사건ㆍ계약을 식별할 수 있는 실제 정보가 없는 문서
- 문체, 목차, 서식, 문장 구조만 참고해도 보안상 위험이 낮은 문서

### 활용 방식

- 비식별 샘플 후보로 검토할 수 있습니다.
- 문체, 목차, 문장 구조, 항목 배열을 참고할 수 있습니다.
- 실제 원문이 아니라 일반화된 형식과 패턴만 기록합니다.

### 예시 유형

- 공개 가능한 안내문 형식
- 일반적인 요청문 문장 구조
- placeholder만 포함된 예시 양식
- 외부 공개 자료 수준의 목차 구성

### 주의사항

A등급으로 보이더라도 실제 파일명, 실제 문서명, 실제 기관명, 실제 담당자명은 프로젝트 문서에 그대로 기록하지 않습니다.

공개 문서라 하더라도 Custom GPT Knowledge 후보로 사용하기 전에는 개인정보, 내부 운영정보, 비공개 정보가 포함되어 있지 않은지 최종 검수합니다.

공개 가능 여부가 애매한 경우에는 A등급으로 분류하지 않고 B등급 이상으로 보수 검토합니다.

## B등급: 비식별 후 제한적 활용 가능

### 기준

- 개인정보 또는 민감정보가 문서의 핵심 내용은 아니며, 기관명, 부서명, 담당자 역할, 날짜, 문서번호 등 일반 식별정보를 placeholder로 치환하면 문체ㆍ구조만 남길 수 있는 문서
- 실제 원문 내용이 아니라 문장 구조, 항목 순서, 표현 방식만 추출할 수 있는 문서
- 비식별 처리 후에도 개인, 사건, 계약, 내부 판단이 특정되지 않는 문서
- A등급보다 보수적으로 관리해야 하지만, C/D등급 사유가 명확하지 않은 문서

### 활용 방식

- 실제 값은 모두 placeholder로 치환합니다.
- 원문 문장, 원문 요약, 실제 맥락을 기록하지 않습니다.
- 문체, 구조, 항목명, 번호 체계, 반복 표현만 일반화해 검토합니다.
- A/B등급 문서만 비식별 샘플 후보로 검토합니다.

### 예시 유형

- 일반화 가능한 요청문 구조
- 실제 식별정보를 제거하면 남길 수 있는 업무 안내 형식
- 항목 배열과 문체만 참고 가능한 서식 유형
- 개인 또는 사건 식별정보 없이 구조만 추출 가능한 문장 패턴

### 주의사항

기관명, 담당자명, 날짜, 문서번호, 수신처, 업체명, 금액 등은 placeholder로 치환해야 합니다.

민원인 실명, 연락처, 주소, 민원번호, 접수번호, 사건번호 등 개인 또는 사건을 직접 식별할 수 있는 정보가 포함된 경우에는 B등급으로 분류하지 않고 C 또는 D등급으로 보수 검토합니다.

비식별 후에도 특정 기관, 개인, 업체, 사건, 계약, 내부 판단을 추정할 수 있으면 C 또는 D등급으로 상향합니다.

## C등급: 내부 참고만 가능

### 기준

- 내부 운영정보, 검토 절차, 업무 판단 기준이 포함될 가능성이 있는 문서
- 외부 AI에 입력하기 부적절하지만 내부 승인 환경에서는 참고 검토가 가능한 문서
- 비식별 처리 후에도 내부 업무 흐름이나 판단 기준이 드러날 수 있는 문서
- 특정 개인이나 사건을 직접 식별하지 않더라도 내부 맥락이 민감한 문서

### 활용 방식

- 외부 AI 처리에 사용하지 않습니다.
- 외부 AI 입력 또는 외부 AI 학습ㆍ참고자료로 사용하지 않습니다.
- Custom GPT Knowledge 후보에서 제외합니다.
- 비식별 샘플 후보로 사용하지 않습니다.
- 원문, 원문 요약, 실제 맥락을 저장소에 기록하지 않습니다.
- 내부 기준 확인 후 패턴화 가능 여부만 별도로 판단합니다.

### 예시 유형

- 내부 검토 절차가 드러나는 문서
- 비공개 업무 흐름이 포함된 자료
- 내부 기준 또는 운영 판단이 포함된 자료
- 외부 공개 여부가 불명확한 업무 참고 자료

### 주의사항

C등급 문서는 내부에서 목차, 표 구조, 결재 흐름 등만 수동으로 참고할 수 있습니다.

C등급 문서를 B등급으로 낮추려면 개인정보 제거만으로는 부족합니다. 업무 맥락, 내부 판단, 업체 정보, 예산 정보가 외부화되어도 안전한지 별도 검토가 필요합니다.

C등급 문서는 Custom GPT Knowledge 후보와 비식별 문체 샘플 후보에서 제외하는 것을 원칙으로 합니다.

## D등급: 외부 처리 금지

### 기준

- 개인정보, 민감정보, 민원ㆍ사건 식별정보가 포함된 문서
- 대외비, 비공개 자료, 보안 관련 세부 내용이 포함될 가능성이 높은 문서
- 감사, 징계, 수사, 조사, 분쟁, 민원 처리와 관련된 실제 세부 내용이 포함된 문서
- 특정 업체 평가, 선정, 계약 판단 등 내부 의사결정에 영향을 줄 수 있는 정보가 포함된 문서
- 비식별 처리로도 개인, 사건, 기관, 업체, 계약, 내부 판단을 충분히 분리하기 어려운 문서

### 활용 방식

- 외부 AI 입력을 금지합니다.
- 비식별 샘플 후보에서 제외합니다.
- 패턴 추출 대상에서도 제외합니다.
- 원문, 원문 요약, 실제 맥락, 실제 식별정보를 저장소에 기록하지 않습니다.

### 예시 유형

- 민원 또는 사건 식별정보가 포함된 자료
- 개인정보 또는 민감정보가 포함된 자료
- 실제 계약, 평가, 선정 판단과 관련된 자료
- 비공개 내부 판단 또는 보안 관련 자료

### 주의사항

D등급 문서는 개인정보 일부를 마스킹하더라도 외부 AI 활용 대상으로 전환하지 않습니다.

문서 맥락 자체가 민감하면 비식별 샘플 작성 대상에서도 제외합니다.

D등급 문서는 Git 프로젝트에 원문, 요약, 실제 파일명, 실제 문서명, 실제 식별정보를 기록하지 않습니다.

D등급 문서는 Custom GPT Knowledge 후보, API 테스트 자료, 자동화 테스트 자료로 사용하지 않습니다.

## 자동 D등급 처리 기준

아래 항목에 해당하는 경우에는 기본적으로 D등급으로 분류합니다. 단, 일부 항목은 최소 C등급 이상으로 분류한 뒤 실제 포함 정보의 민감도에 따라 D등급 여부를 판단합니다.

| 포함 정보 유형 | 판단 기준 |
|---|---|
| 주민등록번호, 여권번호 등 고유식별정보 | D등급 |
| 계좌번호, 카드번호, 금융 거래정보 | D등급 |
| 건강정보, 장애정보, 민감한 신상정보 | D등급 |
| 민원인 실명, 연락처, 주소, 민원번호, 접수번호, 사건번호 | D등급 |
| 차량번호, 위치정보, 현장 식별정보 | D등급 |
| 감사, 징계, 수사, 조사, 분쟁 관련 실제 세부 내용 | D등급 |
| 비공개 내부 검토의견 또는 의사결정 과정 | D등급 |
| 실제 업체 견적 금액, 계약 조건, 평가 의견 | 최소 C등급 이상으로 분류하고, 특정 업체 평가ㆍ선정ㆍ계약 판단이 포함된 경우 D등급 |
| 보안 설정, 접근 권한, 내부 시스템 운영정보 | D등급 |

## 등급 판단 순서

1. 실제 개인정보 또는 민감정보가 포함되어 있는지 확인합니다.
2. 민원, 사건, 접수, 계약, 평가, 내부 판단을 직접 식별할 수 있는지 확인합니다.
3. 비식별 처리 후에도 특정 개인, 기관, 업체, 사건, 계약을 추정할 수 있는지 확인합니다.
4. 외부 AI 처리에 사용해도 되는 공개 수준의 구조인지 확인합니다.
5. 판단이 애매하면 더 보수적인 등급으로 분류합니다.

판단이 애매한 경우에는 더 보수적인 등급으로 분류합니다.

- A인지 B인지 애매하면 B등급으로 분류합니다.
- B인지 C인지 애매하면 C등급으로 분류합니다.
- C인지 D인지 애매하면 D등급으로 분류합니다.
- 등급 판단이 어려운 문서는 비식별 샘플 후보로 사용하지 않고 별도 검토 대상으로 둡니다.

## 문서 인벤토리 기록 방식

문서 인벤토리에는 실제 파일명, 실제 문서명, 실제 기관명, 실제 담당자명, 실제 문서번호, 실제 개인정보, 실제 내부 운영정보를 기록하지 않습니다.

인벤토리에는 아래 수준의 일반화된 정보만 기록합니다.

- 문서 유형
- 예상 보안등급
- 활용 가능 여부
- 비식별 샘플 후보 여부
- 제외 사유의 일반화된 분류
- 추가 검토 필요 여부

실제 파일명과 DOC 코드 매칭표가 필요한 경우, 해당 매칭표는 Git 프로젝트 안에 저장하지 않고 로컬 비공개 메모 또는 내부 승인된 보관 위치에만 별도 보관합니다.

Git 프로젝트에는 DOC 코드, 문서유형, 보안등급, AI 활용 가능 여부, 판단 사유만 기록합니다.

매칭표에는 실제 파일명, 실제 문서명, 실제 보관 위치 등이 포함될 수 있으므로 Custom GPT, ChatGPT, Codex, 외부 AI, GitHub 저장소에 입력하거나 업로드하지 않습니다.

## 기록 금지 항목

아래 항목은 이 문서와 문서 인벤토리에 기록하지 않습니다.

- 실제 공문 원문
- 실제 파일명 또는 실제 문서명
- 실제 기관명, 관서명, 업체명, 담당자명
- 실제 전화번호, 이메일, 주소
- 실제 문서번호, 민원번호, 접수번호, 사건번호, 차량번호
- 실제 업체 견적 금액, 계약 조건, 내부 검토의견
- 실제 원문 요약 또는 실제 내부 운영정보

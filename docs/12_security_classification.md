# 보안 분류 기준

## 목적

문서와 업무 지시를 안전하게 다루기 위해 보안 분류 기준을 정의합니다.

## 분류 단계

| 등급 | 의미 | 처리 방식 |
|---|---|---|
| 공개 가능 | 일반적인 문서 구조나 작성요령 | 비식별 상태에서 초안 작성 가능 |
| 내부 검토 | 내부 기준 확인이 필요한 업무 문서 | 실제 세부자료 없이 구조만 정리 |
| 민감 | 개인정보나 내부 운영정보 가능성 있음 | 비식별화 후 제한적으로 패턴만 검토 |
| 제한 | 대외비, 비공개, 실제 민원ㆍ감사ㆍ수사ㆍ징계 등 | 외부 AI 처리 대상에서 제외 |

## 공개 가능 예시

- 공문 기본 목차
- 조사표 일반 항목
- 업체 견적 요청 메일의 일반 구조
- 추진계획서 목차

## 내부 검토 예시

- 관서별 현황표 구조
- 장비 관리 조사표 구조
- 내부 승인 절차가 필요한 검토자료 목차

## 민감 예시

- 민원인 관련 정보
- 직원 또는 담당자 실명
- 실제 차량 식별정보
- 실제 내부 시스템명
- 실제 운영 수량

## 제한 예시

- 대외비 문서 원문
- 비공개 내부자료
- 실제 감사ㆍ징계ㆍ수사 관련 세부자료
- 실제 민원자료 원문
- 실제 업체 견적자료 원문

## 판단 원칙

- 애매하면 더 높은 보안 등급으로 분류합니다.
- 실제 값은 기록하지 않고 placeholder로 표시합니다.
- 외부 AI에는 비식별화된 업무 지시만 사용합니다.
- 내부 승인 환경에서만 다뤄야 하는 정보는 `[내부 기준에 따라 확인 필요]`로 표시합니다.

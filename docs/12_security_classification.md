# 보안 분류 기준

## 목적

3단계 문서 패턴 추출 작업에서 기존 문서를 어떤 수준까지 활용할 수 있는지 판단하기 위한 A/B/C/D 보안등급을 정의합니다.

## 보안등급 요약

| 등급 | 기준 | 처리 방법 |
|---|---|---|
| A | 공개 가능 | 비식별 샘플 또는 패턴 문서에 사용 가능 |
| B | 비식별 후 사용 가능 | 개인정보ㆍ기관정보ㆍ내부정보 제거 후 패턴만 사용 |
| C | 내부 참고만 가능 | 외부 AI 처리 금지, 내부 승인 환경에서만 참고 |
| D | 외부 처리 금지 | 패턴 추출 대상에서도 제외 |

## A등급: 공개 가능

### 기준

- 공개 가능한 일반 문서 구조
- 개인정보나 내부 운영정보가 없음
- 문체ㆍ서식ㆍ목차만 참고해도 위험이 낮음

### 예시

- `[문서 A]`: 일반 공문 목차 구조
- `[문서 B]`: 공개 가능한 작성요령
- `[문서 C]`: placeholder 기반 업체 메일 양식

### 처리 방법

- 비식별 샘플 세트에 포함 가능
- Custom GPT 테스트용 문체 샘플 후보 가능

## B등급: 비식별 후 사용 가능

### 기준

- 일부 기관명, 담당자명, 업무 배경, 내부 항목이 포함될 가능성이 있음
- 실제 내용 제거 후 문장 구조만 남길 수 있음

### 예시

- `[문서 D]`: 현황조사 공문 구조
- `[문서 E]`: 추진계획서 목차
- `[문서 F]`: 자료 제출 요청 문장 패턴

### 처리 방법

- 실제 값은 모두 placeholder로 치환
- 원문 요약 금지
- 문체ㆍ구조ㆍ번호 체계만 추출

## C등급: 내부 참고만 가능

### 기준

- 내부 운영정보 가능성이 있음
- 외부 AI에 입력하기 부적절함
- 내부 승인 환경에서만 검토해야 함

### 예시

- `[문서 G]`: 내부 현황자료
- `[문서 H]`: 내부 승인 절차가 포함된 자료
- `[문서 I]`: 세부 운영 기준이 포함된 자료

### 처리 방법

- 외부 AI 처리 금지
- 저장소에 원문 또는 요약을 기록하지 않음
- 내부 기준 확인 후 패턴화 가능 여부만 판단

## D등급: 외부 처리 금지

### 기준

- 개인정보, 민감정보, 대외비, 비공개 자료 가능성이 높음
- 민원, 감사, 징계, 수사, 보안 관련 세부내용 포함 가능성이 있음
- 실제 업체 견적자료 또는 내부자료 원문에 해당함

### 예시

- `[문서 J]`: 민감정보 포함 가능 문서
- `[문서 K]`: 비공개 내부자료
- `[문서 L]`: 외부 처리 금지 자료

### 처리 방법

- 외부 AI 입력 금지
- Knowledge 후보 제외
- 비식별 샘플 작성에도 사용하지 않음

## 공통 원칙

- 애매하면 더 높은 보안등급으로 분류합니다.
- 실제 기관명이나 실제 사례를 기록하지 않습니다.
- Knowledge 업로드는 3단계 완료 후 별도 검토 대상으로만 다룹니다.

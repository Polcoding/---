# 문서 인벤토리

## 목적

3단계에서는 기존 공문 원문을 AI에 학습시키지 않습니다. 기존 문서에서 문체ㆍ구조ㆍ서식 패턴만 추출할 수 있는지 검토하고, 실제 원문 대신 비식별 샘플 세트를 만들기 위한 목록을 관리합니다.

## 작성 원칙

- 문서명은 실제 파일명 대신 `[문서 A]`, `[문서 B]`처럼 placeholder로 기록합니다.
- 실제 공문 원문, 문서번호, 기관명, 관서명, 담당자명, 수신처, 날짜를 기록하지 않습니다.
- 원문 내용 요약이 아니라 문서 유형과 패턴 추출 가능성만 기록합니다.
- 활용 가능 여부는 보안 분류와 비식별 가능성을 함께 보고 판단합니다.

## 문서 인벤토리 표

| 문서명 | 문서유형 | 용도 | 포함 가능 정보 | 민감정보 가능성 | 활용 가능 여부 | 비고 |
|---|---|---|---|---|---|---|
| [문서 A] | [공문/기획서/조사표/메일 등] | [용도 확인 필요] | [서식ㆍ문체ㆍ구조] | [낮음/중간/높음/확인 필요] | [가능/비식별 후 가능/내부 참고/제외] | [비고] |
| [문서 B] | [문서유형 확인 필요] | [용도 확인 필요] | [확인 필요] | [확인 필요] | [확인 필요] | [비고] |

## 문서유형 예시

- 공문
- 추진계획서
- 결과보고서
- 현황조사 공문
- 조사표
- 업체 연락 메일
- 검토보고서

## 활용 가능 여부 기준

| 값 | 의미 |
|---|---|
| 가능 | 공개 가능한 문체ㆍ구조만 포함 |
| 비식별 후 가능 | 일부 민감 가능성이 있어 placeholder 치환 필요 |
| 내부 참고 | 외부 AI 처리에는 부적절하나 내부 기준 정리에는 참고 가능 |
| 제외 | 실제 원문, 개인정보, 내부 운영정보, 대외비 가능성이 높음 |

## 기록 금지 항목

- 실제 문서명 또는 실제 파일명
- 실제 기관명 또는 관서명
- 실제 사람 이름
- 실제 차량번호
- 실제 민원번호 또는 접수번호
- 실제 업체명
- 실제 내부 운영정보
- 실제 공문 원문 또는 원문 요약

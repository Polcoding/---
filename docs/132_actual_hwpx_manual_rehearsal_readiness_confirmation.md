# 실제 양식 수동 리허설 준비 확인 결과

## 목적

이 문서는 실제 양식 수동 리허설 전 사용자 확인 게이트 통과 사실을 기록합니다.

이 기록은 실제 기관 HWPX/HWP 원본, 실제 문서 원문, 실제 개인정보, 실제 파일명, 실제 내부 경로를 저장소에 남기기 위한 문서가 아닙니다.

후속 상태는 `docs/134_actual_hwpx_manual_rehearsal_no_copy_hold.md`를 우선합니다. 이후 실제 preview에 사용할 비식별 HWPX 작업 복사본이 없다고 확인되어, 실제 양식 수동 리허설은 보류 상태로 전환했습니다.

## 확인 근거

사용자는 실제 양식 수동 리허설 전 다음 항목을 모두 확인했다고 명시했습니다.

사용자 확인 표현:

```text
1~3 모두 올클리어
```

## 사용자 확인 완료 항목

| 상태 | 확인 항목 | 의미 |
|---|---|---|
| 확인 완료 | 실제 원본과 작업 복사본이 저장소 밖에만 있음 | 저장소 안에 실제 HWP/HWPX 원본 또는 작업 복사본을 두지 않음 |
| 확인 완료 | 실제 본문, 기관명, 문서번호, 결재선, 직인, 로고, 담당자 정보 제거됨 | placeholder 후보로 전환하기 전 식별 요소를 제거함 |
| 확인 완료 | GitHub Desktop Changes에 HWP/HWPX 원본 또는 Git 추적 대상 output 파일이 없음 | Git 추적 대상에 실제 양식이나 로컬 산출물이 보이지 않음 |

## Codex가 확인한 범위

Codex가 확인할 수 있는 범위는 저장소 내부 상태와 Git 제외 규칙입니다.

확인 가능한 항목:

- 저장소 작업 트리 상태
- `renderers/hwpx_renderer/output/` 산출물 Git 제외 상태
- `normalizers/output/` 산출물 Git 제외 상태
- `templates/hwpx/*.hwpx` 로컬 템플릿 Git 제외 상태

Codex가 직접 확인할 수 없는 항목:

- 저장소 밖 실제 원본 위치
- 저장소 밖 작업 복사본 내용
- 한컴 문서 속성, 미리보기, 마지막 저장자 정보
- 실제 기관명, 결재선, 직인, 로고 제거 여부
- GitHub Desktop 화면의 사용자 실제 표시 상태

따라서 이 문서는 사용자의 확인 선언과 저장소 내부 검증을 함께 기록하되, 실제 원본 검토를 Codex가 대신 수행했다는 의미가 아닙니다.

## 다음 단계 진입 판단

이 문서 작성 시점의 사용자 확인 기준으로는 실제 양식 수동 리허설 결과를 gap log 형식으로 기록하는 단계로 넘어갈 수 있다고 판단했습니다. 다만 후속 상태에서는 실제 preview에 사용할 비식별 HWPX 작업 복사본 없음이 확인되어 수동 리허설을 보류합니다.

진입 가능한 범위:

- 저장소 밖 작업 복사본을 사용자가 한컴에서 직접 열람
- 실제 원문 없이 문서유형, 항목, 증상, 심각도, 후속 조치만 기록
- 표가 포함된 문서는 `table_scope: frame_only` 기준으로 표 위치, 폭, 줄바꿈, 겹침, 여백만 확인
- gap이 없으면 문서유형별 `문제 없음`으로 기록
- gap이 있으면 실제 문구 없이 증상 중심으로 기록

계속 보류하는 범위:

- 실제 기관 HWPX/HWP 원본 Git 추가
- 실제 업무용 HWPX output 재생성
- 실제 원문, 실제 기관명, 실제 문서번호 기록
- renderer, normalizer, fixture, routing, HWPX payload 변경
- HWPX 표 내부 실제 데이터 자동 입력 또는 Excel/한셀 자동 연동
- style profile 미확정값 임의 확정
- Email, API, Make.com 실제 연동 구현

## gap log 제공 형식

사용자가 다음 단계에서 제공할 수 있는 형식은 아래와 같습니다.

```text
document_type: one_page_report
section: 1. 보고 개요
gap_type: 글자 겹침
severity: 수정 필요
observed_symptom: 본문 첫 행과 다음 행 간격이 좁음
expected_state: 행 간격이 겹치지 않고 구분되어야 함
suspected_scope: 템플릿 문단 배치
next_action: 템플릿 후보 점검
```

문제가 없을 때는 아래처럼 기록합니다.

```text
document_type: one_page_report
status: 문제 없음
```

표가 포함된 문서도 문제가 없으면 아래처럼만 기록합니다.

```text
document_type: one_page_report
table_scope: frame_only
status: 문제 없음
```

## 기록 금지

gap log와 후속 문서에는 다음 값을 기록하지 않습니다.

- 실제 원문 일부
- 실제 파일명
- 실제 내부 경로
- 실제 기관명
- 실제 부서명
- 실제 담당자명
- 실제 문서번호
- 실제 결재선, 직인, 로고 설명
- 실제 예산, 일정, 실적 수치
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록
- 실제 개인정보 또는 민감정보

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- 실제 파일명, 내부 경로, 원본 양식명 추가 없음
- 폰트 파일 추가 없음
- API 키, 토큰, 인증정보 예시값 추가 없음
- Email, API, Make.com 연동 구현 없음

## 결론

사용자 확인 기준으로 실제 양식 수동 리허설 전 준비 게이트는 통과한 것으로 기록했습니다.

후속 기준의 다음 단계는 실제 파일을 저장소에 추가하는 것이 아니라, 사용자가 저장소 밖 비식별 HWPX 작업 복사본 준비를 다시 명시했을 때 `docs/150_manual_preview_resume_gate.md` 기준으로 재개 조건을 확인하는 것입니다.

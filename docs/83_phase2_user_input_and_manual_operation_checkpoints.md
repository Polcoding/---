# Phase 2 사용자 입력 및 수동 운영 점검표

## 목적

Phase 2 최소 운영 흐름에서 사용자가 입력 전, HWPX output 생성 전, HWPX output 생성 후 확인해야 할 항목을 정리합니다.

이 문서는 실제 원문을 처리하기 위한 양식이 아니라, 비식별 placeholder 기반 초안 생성 흐름을 안전하게 반복하기 위한 점검 기준입니다.

## 입력 전 사용자 체크리스트

사용자는 업무 지시를 넣기 전에 다음을 확인합니다.

| 확인 | 항목 | 기준 |
|---|---|---|
| [ ] | 실제 공문 원문 제외 | 원문 전체를 붙여넣지 않음 |
| [ ] | 실제 개인정보 제외 | 이름, 연락처, 이메일, 주소, 고유식별정보 제외 |
| [ ] | 실제 기관명ㆍ부서명 제외 | `[기관명]`, `[부서명]` 등 placeholder 사용 |
| [ ] | 실제 문서번호 제외 | 실제 문서번호, 접수번호, 민원번호, 사건번호 제외 |
| [ ] | 실제 예산ㆍ계약값 제외 | 금액, 견적, 업체 평가, 계약 조건 제외 |
| [ ] | 실제 내부 운영정보 제외 | 내부 시스템명, 계정, 권한, 차량번호, 장비 세부값 제외 |
| [ ] | 문서 유형 표시 | `one_page_report`, `project_plan`, `result_report`, `review_report` 중 하나로 표현 |
| [ ] | 확인 필요값 분리 | 일정, 예산, 실적, 담당자, 수량은 모르면 `[확인 필요]`로 표시 |
| [ ] | 확인 필요값 의미 확인 | `[확인 필요]`는 실제값이 아니며, 이후 `missing_fields` 검토용 항목으로 분리될 수 있음 |
| [ ] | 자동화 금지 요청 제외 | 발송, 결재, 계약, 예산 집행, 법률 판단 자동화 요청 금지 |

## 권장 입력 형태

```text
[비식별 업무] [문서 유형] 초안 작성.
목적: [확인 필요]
배경: [비식별 배경 요약]
주요 내용: [확인 필요]
일정/예산/실적/담당자: [확인 필요]
민감정보 또는 실제 원문 없음.
```

## 입력 예시

허용 가능한 예:

```text
[비식별 사업] 추진계획서 초안 작성.
목적은 [확인 필요]이며, 추진 배경은 [비식별 배경 요약]입니다.
일정, 예산, 담당자는 아직 확정되지 않아 [확인 필요]로 유지합니다.
실제 개인정보와 실제 문서번호는 포함하지 않았습니다.
```

보류 또는 차단 대상 예:

- 실제 문서 원문을 그대로 넣는 요청
- 실제 담당자명, 연락처, 이메일이 있는 요청
- 실제 문서번호, 민원번호, 사건번호가 있는 요청
- 실제 계약 금액, 업체 평가, 선정 사유가 있는 요청
- "그대로 발송", "자동 결재", "예산 집행 처리" 요청

## HWPX output 생성 전 수동 점검

Codex 또는 사용자는 HWPX output 생성 전에 다음을 확인합니다.

| 확인 | 항목 | 기준 |
|---|---|---|
| [ ] | routing 확인 | `ready_for_draft` 또는 허용된 `needs_more_input`만 렌더링 |
| [ ] | `review_report` 승인 조건 확인 | 승인 없는 `review_report`는 렌더링 금지 |
| [ ] | `missing_fields` 상태 확인 | 누락값은 검토용 목록으로 유지하고 자동 제외하지 않음 |
| [ ] | validation 통과 확인 | `validate_hwpx_payload_poc.py` 통과 |
| [ ] | dry-run 통과 확인 | `hwpx_renderer_dry_run_poc.py` 통과 |
| [ ] | placeholder 템플릿 존재 확인 | 로컬 `templates/hwpx/*.hwpx` 존재 |
| [ ] | Git 제외 확인 | 로컬 HWPX 템플릿과 output이 ignored 상태 |
| [ ] | 민감정보 패턴 확인 | 실제 연락처, 이메일, 문서번호 형식 없음 |

## HWPX output 생성 후 사용자 검수

[사용자 확인 필요]

사용자는 한컴에서 output HWPX를 열고 다음을 확인합니다.

| 확인 | 항목 | 기준 |
|---|---|---|
| [ ] | 파일 열림 | 한컴에서 정상 열림 |
| [ ] | 글자 겹침 | 겹침 없음 |
| [ ] | 항목 순서 | 제목과 1~10번 항목 순서 정상 |
| [ ] | 줄바꿈 | 본문 줄바꿈과 문단 배치 정상 |
| [ ] | bullet 표기 | 내용 앞 `-` 표기 일관성 |
| [ ] | placeholder 잔여 | 남은 `{{placeholder}}` 없음 |
| [ ] | 확인 필요값 | `[확인 필요]`가 실제값처럼 오인되지 않음 |
| [ ] | `missing_fields` 표시 | HWPX 본문과 검토용 확인 목록이 분리되어 보임 |
| [ ] | 검토용 목록 | `missing_fields`를 본문 실제값으로 확정하지 않음 |
| [ ] | 민감정보 | 실제 개인정보, 기관정보, 문서번호 없음 |

## 이상 발견 시 처리

| 상황 | 처리 |
|---|---|
| 글자 겹침 | 렌더러 formatting 또는 로컬 템플릿 문단 구조 수정 |
| `{{placeholder}}` 잔여 | placeholder map 또는 템플릿 placeholder 오탈자 확인 |
| `missing_fields` 실제값 확정 | 즉시 중단하고 검토용 목록으로 되돌림 |
| 실제값 의심 | 즉시 중단하고 비식별 입력으로 재작성 |
| `review_report` 승인 누락 | 렌더링 금지, 사람 보안 검토 후 재시도 |
| GitHub Desktop Changes에 HWPX 표시 | 즉시 중단하고 `.gitignore` 및 파일 위치 확인 |

# HWPX 보고서 4종 사용자 입력 템플릿

## 목적

Phase 2 최소 운영 흐름에서 사용자가 HWPX 보고서 초안 생성을 요청할 때 사용할 placeholder 기반 입력 템플릿을 정리합니다.

이 문서는 실제 원문, 실제 개인정보, 실제 기관정보를 넣기 위한 양식이 아닙니다. 사용자는 실제값을 제거한 뒤 `[확인 필요]` 또는 placeholder로 남겨야 합니다.

## 빠른 사용 순서

사용자는 아래 순서만 따라가면 됩니다.

1. 만들 문서 유형을 고릅니다.
2. 해당 문서 유형의 입력 템플릿을 사용합니다.
3. 실제값, 개인정보, 기관명, 문서번호, 실제 원문을 넣지 않습니다.
4. 모르는 값은 `[확인 필요]`로 둡니다.
5. 표가 필요할 것 같으면 `[표 데이터 별도 확인 필요]`로 표시합니다.
6. Codex는 입력값을 바탕으로 placeholder 초안과 확인 필요 목록을 정리합니다.
7. 실제 HWPX 양식, 글자 겹침, 표 폭, 여백은 한컴에서 사람이 확인합니다.

## 역할 구분

| 표시 | 의미 | 처리 방식 |
|---|---|---|
| `[사용자 확인 필요]` | 사람이 직접 판단하거나 확인해야 하는 값 | 실제값을 바로 넣지 말고 placeholder 유지 또는 내부 검토 후 처리 |
| `[Codex 처리 가능]` | 비식별 placeholder 입력을 바탕으로 구조화 가능한 항목 | 초안 문장, 개조식 정리, missing_fields 표시, 보안 주의문구 작성 |
| `[보류]` | 현재 단계에서 자동화하지 않는 항목 | 실제 HWPX 원본, 실제 표 데이터, Excel/한셀 연동, Email/API/Make.com 연동 |

## 문서 유형별 빠른 확인표

| 문서 유형 | 사용자가 먼저 확인할 것 | Codex가 처리할 수 있는 것 | 현재 보류 |
|---|---|---|---|
| `one_page_report` | `[사용자 확인 필요]` 보고 목적, 주요 내용, 향후 조치 | `[Codex 처리 가능]` 원페이지 보고서 개조식 초안, 누락값 목록 | `[보류]` 실제 양식 서식값 확정 |
| `project_plan` | `[사용자 확인 필요]` 추진 일정, 예산, 추진대상, 추진방식 | `[Codex 처리 가능]` 추진계획서 목차와 단계별 초안 | `[보류]` 실제 일정표ㆍ예산표 자동 작성 |
| `result_report` | `[사용자 확인 필요]` 실적, 결과 수치, 계획 대비 결과 | `[Codex 처리 가능]` 계획 대비 결과 구조화, 확인 필요 항목 표시 | `[보류]` 실제 실적표ㆍ수량ㆍ금액 자동 입력 |
| `review_report` | `[사용자 확인 필요]` 검토 범위, 필요 검토 주체, 보안 검토 승인 여부 | `[Codex 처리 가능]` 보안 승인 조건이 있는 placeholder 검토보고 초안 | `[보류]` 실제 검토의견 원문 처리 |

`[사용자 확인 필요]` 항목은 Codex가 임의로 채우지 않습니다.

## 공통 입력 원칙

- 실제 공문 원문을 붙여넣지 않습니다.
- 실제 개인정보, 기관명, 부서명, 담당자명, 연락처, 이메일을 넣지 않습니다.
- 실제 문서번호, 접수번호, 민원번호, 사건번호를 넣지 않습니다.
- 실제 예산액, 계약 금액, 업체 평가, 선정 사유를 넣지 않습니다.
- 일정, 예산, 실적, 담당자, 수량이 불명확하면 `[확인 필요]`로 둡니다.
- AI는 초안만 만들며 발송, 결재, 계약, 예산 집행, 법률 판단은 하지 않습니다.

## 표 데이터 후보 표시 원칙

표 데이터 후보는 실제 표 데이터가 아니라, 향후 Excel/한셀 분리 검토가 필요한지 표시하는 사용자 확인 항목입니다.

- 기본 표현은 `[표 데이터 별도 확인 필요]`로 둡니다.
- 개별 수량, 금액, 대상 목록, 일정은 `[확인 필요]`로 둡니다.
- 표 데이터가 필요하지 않다고 사람이 확인한 경우에만 `해당 없음`으로 바꿀 수 있습니다.
- 사용자 입력 템플릿에는 `null`만 단독으로 쓰지 않습니다.
- 표 데이터 후보를 이유로 HWPX payload, renderer, normalizer, fixture, routing을 변경하지 않습니다.
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록은 입력하지 않습니다.

권장 공통 입력 줄:

```text
표 데이터 필요 여부: [확인 필요]
표 데이터 후보: [표 데이터 별도 확인 필요]
수량ㆍ금액ㆍ대상 목록: [확인 필요]
Excel/한셀 분리 후보: [확인 필요]
```

## review_report 입력 템플릿

`review_report`는 기본적으로 보안 검토가 필요한 문서 유형입니다.

승인 없는 입력은 `needs_security_review`로 멈춰야 하며, HWPX 렌더링하지 않습니다.

사용자가 먼저 확인할 핵심 항목:

- `[사용자 확인 필요]` 검토 범위
- `[사용자 확인 필요]` 필요 검토 주체
- `[사용자 확인 필요]` placeholder 렌더링 보안 승인 여부

Codex 처리 가능 항목:

- `[Codex 처리 가능]` 승인 조건이 있는 검토보고서 placeholder 초안
- `[Codex 처리 가능]` 위험요소와 후속조치 개조식 정리
- `[Codex 처리 가능]` 보안 검토 필요 상태 표시

현재 보류 항목:

- `[보류]` 실제 검토의견 원문 처리
- `[보류]` 실제 승인자명, 결재번호, 문서번호 입력
- `[보류]` 보안 검토 없는 HWPX 렌더링

```text
[비식별 사안] review_report 초안 작성.

제목 힌트: [비식별 사안 검토보고]
검토 배경: [비식별 배경 요약]
검토 범위: [검토 범위 확인 필요]
검토 항목:
1. [검토 항목 확인 필요]
2. [검토 항목 확인 필요]
검토 의견: [검토 의견 확인 필요]
위험요소:
1. [위험요소 확인 필요]
2. [추가 검토 필요사항 확인 필요]
필요 검토: [법무ㆍ계약ㆍ개인정보 검토 필요 여부 확인]
표 데이터 후보: [검토항목 목록ㆍ위험요소 매트릭스 확인 필요 또는 해당 없음]
후속조치: [후속조치 확인 필요]

실제 원문, 개인정보, 기관명, 문서번호, 실제 검토의견, 승인자명, 결재번호는 포함하지 않았습니다.
확인되지 않은 값은 [확인 필요]로 유지해 주세요.
```

### review_report placeholder 렌더링 승인 신호

검토보고서를 HWPX로 렌더링하려면 사람이 보안 검토를 마쳤다는 placeholder 승인 신호가 별도로 필요합니다.

```json
{
  "security_gate": {
    "human_security_review_completed": true,
    "review_basis": "[비식별 보안 검토 완료]",
    "approved_for_placeholder_rendering": true
  }
}
```

이 신호에는 실제 승인자명, 실제 결재번호, 실제 검토의견 원문을 넣지 않습니다.

## 금지 입력 예시

다음 유형은 입력하지 않습니다.

```text
실제 공문 원문을 붙여넣고 그대로 보고서로 만들어 주세요.
```

```text
실제 담당자명, 연락처, 이메일, 문서번호를 포함해 작성해 주세요.
```

```text
업체 선정 사유와 계약 금액을 확정해서 넣어 주세요.
```

```text
작성한 문서를 자동 발송하거나 결재까지 처리해 주세요.
```

## 완료 기준

문서 유형별 입력 템플릿은 다음 조건을 만족해야 합니다.

- 실제값 대신 placeholder 또는 `[확인 필요]` 사용
- HWPX 보고서 4종 우선순위 유지
- `review_report` 보안 검토 조건 분리
- 표 데이터 후보를 실제값 없이 별도 확인 항목으로 표시
- 자동 발송, 결재, 계약, 예산 집행, 법률 판단 금지 유지
- 실제 원문, 개인정보, 기관정보 미사용

# HWPX 보고서 4종 사용자 입력 템플릿

## 목적

Phase 2 최소 운영 흐름에서 사용자가 HWPX 보고서 초안 생성을 요청할 때 사용할 placeholder 기반 입력 템플릿을 정리합니다.

이 문서는 실제 원문, 실제 개인정보, 실제 기관정보를 넣기 위한 양식이 아닙니다. 사용자는 실제값을 제거한 뒤 `[확인 필요]` 또는 placeholder로 남겨야 합니다.

## 공통 입력 원칙

- 실제 공문 원문을 붙여넣지 않습니다.
- 실제 개인정보, 기관명, 부서명, 담당자명, 연락처, 이메일을 넣지 않습니다.
- 실제 문서번호, 접수번호, 민원번호, 사건번호를 넣지 않습니다.
- 실제 예산액, 계약 금액, 업체 평가, 선정 사유를 넣지 않습니다.
- 일정, 예산, 실적, 담당자, 수량이 불명확하면 `[확인 필요]`로 둡니다.
- AI는 초안만 만들며 발송, 결재, 계약, 예산 집행, 법률 판단은 하지 않습니다.

## review_report 입력 템플릿

`review_report`는 기본적으로 보안 검토가 필요한 문서 유형입니다.

승인 없는 입력은 `needs_security_review`로 멈춰야 하며, HWPX 렌더링하지 않습니다.

```text
[비식별 사안] review_report 초안 작성.

제목 힌트: [비식별 사안 검토보고]
검토 배경: [비식별 배경 요약]
검토 범위: [검토 범위 확인 필요]
검토 항목:
1. [검토 항목 확인 필요]
2. [검토 항목 확인 필요]
검토 의견: [검토 의견 확인 필요]
위험요소:
1. [위험요소 확인 필요]
2. [추가 검토 필요사항 확인 필요]
필요 검토: [법무ㆍ계약ㆍ개인정보 검토 필요 여부 확인]
후속조치: [후속조치 확인 필요]

실제 원문, 개인정보, 기관명, 문서번호, 실제 검토의견, 승인자명, 결재번호는 포함하지 않았습니다.
확인되지 않은 값은 [확인 필요]로 유지해 주세요.
```

### review_report placeholder 렌더링 승인 신호

검토보고서를 HWPX로 렌더링하려면 사람이 보안 검토를 마쳤다는 placeholder 승인 신호가 별도로 필요합니다.

```json
{
  "security_gate": {
    "human_security_review_completed": true,
    "review_basis": "[비식별 보안 검토 완료]",
    "approved_for_placeholder_rendering": true
  }
}
```

이 신호에는 실제 승인자명, 실제 결재번호, 실제 검토의견 원문을 넣지 않습니다.

## 금지 입력 예시

다음 유형은 입력하지 않습니다.

```text
실제 공문 원문을 붙여넣고 그대로 보고서로 만들어 주세요.
```

```text
실제 담당자명, 연락처, 이메일, 문서번호를 포함해 작성해 주세요.
```

```text
업체 선정 사유와 계약 금액을 확정해서 넣어 주세요.
```

```text
작성한 문서를 자동 발송하거나 결재까지 처리해 주세요.
```

## 완료 기준

문서 유형별 입력 템플릿은 다음 조건을 만족해야 합니다.

- 실제값 대신 placeholder 또는 `[확인 필요]` 사용
- HWPX 보고서 4종 우선순위 유지
- `review_report` 보안 검토 조건 분리
- 자동 발송, 결재, 계약, 예산 집행, 법률 판단 금지 유지
- 실제 원문, 개인정보, 기관정보 미사용

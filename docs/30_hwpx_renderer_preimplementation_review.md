# HWPX 렌더러 PoC 구현 전 재검수

## 목적

HWPX 렌더러 PoC 구현 전에 단순 파일 생성 가능성뿐 아니라, 공공기관 문서로 실제 사용할 수 있는 수준의 서식ㆍ문체ㆍ번호체계ㆍ표 양식ㆍ붙임 형식 유지 가능성을 검토합니다.

이번 단계에서는 실제 HWPX 파일이나 코드를 생성하지 않습니다.

실제 공문 원문, 실제 기관명, 실제 업체명, 실제 담당자명, 실제 문서번호, 실제 민원번호, 실제 차량번호, 실제 장비명, 실제 수량, 실제 금액은 사용하지 않습니다.

## 핵심 검토 관점

- HWPX 파일 생성 가능성
- 기관 표준 서식 유지 가능성
- 폰트, 자간, 줄간격, 문단 간격 유지 가능성
- 공문 번호체계 유지 가능성
- 표 양식 유지 가능성
- 붙임 표기와 끝. 위치 유지 가능성
- 공문체 문장 품질 반영 가능성
- 개인정보ㆍ민감정보 차단 가능성
- 사람이 최종 검토 가능한 초안 생성 가능성

## HWPX 최소 PoC 범위

최소 PoC는 실제 기관 양식이나 실제 공문 원문을 사용하지 않고 placeholder 기반 샘플만 사용합니다.

- 입력: placeholder 기반 공문 JSON 샘플
- 출력: 실제 HWPX 파일 생성 전 설계 검토 또는 별도 승인 후 로컬 테스트 산출물
- 대상 문서: 일반 공문 또는 현황조사 지시 공문 1종
- 필수 영역: 상단 정보 placeholder, 제목, 본문, 붙임, 끝 표시
- 번호체계: 1. / 가. / 1) / 가) 유지 여부 확인
- 붙임: 붙임이 0개인 경우와 여러 개인 경우의 표시 방식 확인
- 표: 단순 표 1개 이하로 제한하고 셀 병합ㆍ테두리 유지 가능성 확인
- 보안: 렌더링 전 risk_level, 개인정보ㆍ민감정보ㆍ내부정보 플래그 차단 확인
- 검토: 생성 결과는 실제 발송물이 아니라 사람이 검토하는 초안으로만 취급

## HWPX 렌더링 전 차단 조건

다음 조건 중 하나라도 해당하면 HWPX 렌더링을 중단하거나 사람 검토 대상으로 전환합니다.

- security_review.risk_level이 high 또는 blocked
- contains_personal_info가 true
- contains_sensitive_info가 true
- contains_internal_info가 true
- 실제 개인정보처럼 보이는 값 포함
- 실제 문서번호, 민원번호, 접수번호, 사건번호처럼 보이는 값 포함
- 실제 기관명, 관서명, 업체명, 담당자명처럼 보이는 값 포함
- 실제 차량번호, 장비명, 수량, 내부 운영정보처럼 보이는 값 포함
- 실제 예산액, 견적 금액, 계약 조건처럼 보이는 값 포함
- 내부망, 시스템명, 계정, 비밀번호, API 키 관련 값 포함
- 계약, 업체 선정, 발주, 예산 집행이 확정된 것처럼 보이는 표현 포함

## 구현 전 위험요소

| 위험요소 | 설명 | 대응 방향 |
|---|---|---|
| 서식 손상 | HWPX 생성 과정에서 폰트, 자간, 줄간격, 문단 간격이 깨질 수 있음 | 실제 서식값은 [확인 필요]로 두고 템플릿 유지 방식 검증 |
| 번호체계 오류 | 상위 번호와 하위 번호가 섞이거나 잘못된 번호가 생성될 수 있음 | 번호체계 검수 체크리스트 적용 |
| 표 양식 손상 | 셀 병합, 테두리, 셀 여백, 표 머리글이 유지되지 않을 수 있음 | 단순 표부터 PoC 범위 제한 |
| 붙임ㆍ끝 표시 오류 | 붙임 번호, 끝. 위치가 실제 공문체 기준과 달라질 수 있음 | 붙임 0개ㆍ여러 개 시나리오 분리 검증 |
| 문체 부적합 | 구어체, 홍보성 표현, 미확정 사항 단정이 섞일 수 있음 | 공공기관 문체 체크리스트 적용 |
| 보안 위험 | 원문, 개인정보, 내부 운영정보가 렌더링 입력에 포함될 수 있음 | 렌더링 전 차단 조건 적용 |
| 사람 검토 누락 | 초안이 실제 발송 가능한 문서처럼 오해될 수 있음 | 초안 표시와 최종 검토 문구 유지 |

## Step 22 진입 조건

다음 조건이 충족된 뒤에만 Step 22로 진행합니다.

- HWPX 구현 방식 후보의 장단점이 검토되었는가
- 최소 PoC 범위가 일반 공문 또는 현황조사 지시 공문 1종으로 제한되었는가
- HWPX 서식 및 공문체 요구사항 문서가 작성되었는가
- HWPX 서식 충실도 체크리스트가 작성되었는가
- 공공기관 문체 체크리스트가 작성되었는가
- 렌더링 전 보안 차단 조건이 정리되었는가
- 실제 원문, 실제 기관명, 실제 개인정보를 사용하지 않는 원칙이 유지되는가
- 사용자가 명시적으로 HWPX PoC 구현 시작을 지시했는가

## 검토 입력 원칙

- placeholder 기반 JSON 샘플만 사용합니다.
- 실제 공문 원문을 사용하지 않습니다.
- 실제 기관 양식을 저장소에 추가하지 않습니다.
- 실제 개인정보와 내부 운영정보를 입력하지 않습니다.
- 확인되지 않은 서식값은 [확인 필요]로 표시합니다.

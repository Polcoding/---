# 문서 패턴 라이브러리

## 목적

이 문서는 공공기관 행정문서에서 반복적으로 사용할 수 있는 문장 패턴을 placeholder 기반으로 정리하기 위한 안전한 문장 패턴 후보 모음입니다.

기존 공문 원문을 학습시키거나 원문 문장을 옮기는 것이 아니라, 문장 구조와 서식 흐름만 일반화합니다.

이 문서의 패턴은 Custom GPT, 향후 API 프롬프트, 문서 템플릿에서 참고할 수 있는 초안 작성 보조용 후보이며, 실제 업무 적용 전에는 사람이 최종 검토합니다.

## 작성 원칙

- 실제 원문 문장을 복사하지 않습니다.
- 실제 파일명, 문서명, 기관명, 관서명, 업체명, 담당자명을 기록하지 않습니다.
- 실제 개인정보, 민원정보, 내부 운영정보를 기록하지 않습니다.
- 예산, 일정, 담당자, 수량, 법령, 계약 조건은 임의로 만들지 않습니다.
- 필요한 정보는 [확인 필요] 또는 placeholder로 표시합니다.
- 확정되지 않은 내용은 단정하지 않습니다.
- 외부 발송용 문안에는 계약, 선정, 예산 집행이 확정된 것처럼 보이는 표현을 사용하지 않습니다.
- 모든 문장은 초안 작성 보조용입니다.
- A등급 또는 B등급 후보 문서에서 추출 가능한 수준의 일반 문장 패턴만 작성합니다.
- C등급과 D등급 문서의 실제 문체나 문맥은 사용하지 않습니다.
- Custom GPT Knowledge 업로드는 아직 하지 않습니다.
- Make.com, OpenAI API, 이메일 자동화 구현은 하지 않습니다.

## placeholder 공통 규칙

아래 placeholder는 실제 기관명, 실제 사람 이름, 실제 업체명, 실제 날짜, 실제 문서번호처럼 보이지 않도록 사용합니다.

- [업무 발생 배경]
- [대상 기관/부서]
- [조사 대상]
- [조사 기준일]
- [제출기한]
- [제출방법]
- [담당부서]
- [공용 연락처]
- [활용 목적]
- [사업명]
- [행사명]
- [추진기간 확인 필요]
- [소요예산 확인 필요]
- [자료 요청 항목]
- [비식별 처리된 민원 요지]
- [업체 A]
- [계약 또는 선정 미확정 문구]
- [개인정보 제외 안내 문구]

## 1. 공문 기본 문장 패턴

### 1-1. 조사 요청 공문 패턴

조사 목적 문장:

```text
[업무 발생 배경]에 따라 [대상 기관/부서]의 [조사 대상] 현황을 확인하고, 향후 [활용 목적]을 위한 기초자료로 활용하고자 합니다.
```

조사대상 안내 문장:

```text
조사 대상은 [대상 기관/부서]에서 관리하거나 확인 가능한 [조사 대상]으로 하며, 세부 범위는 [확인 필요]로 표시합니다.
```

조사기준일 안내 문장:

```text
작성 기준은 [조사 기준일] 현재 확인 가능한 범위로 하며, 기준일 적용이 어려운 항목은 [확인 필요]로 기재합니다.
```

제출기한 안내 문장:

```text
각 [대상 기관/부서]에서는 작성 자료를 [제출기한]까지 제출하여 주시기 바랍니다.
```

제출방법 안내 문장:

```text
제출 방법은 [제출방법]을 원칙으로 하며, 제출 방식 변경이 필요한 경우 [담당부서]와 사전 협의하여 주시기 바랍니다.
```

작성 유의사항 문장:

```text
작성 시 실제 개인정보, 민감정보, 내부 운영정보가 포함되지 않도록 확인하고, 필요한 경우 [확인 필요]로 표시하여 주시기 바랍니다.
```

붙임 안내 문장:

```text
붙임 서식은 [자료 요청 항목]을 정리하기 위한 참고 양식이며, 실제 식별정보는 포함하지 않습니다.
```

### 1-2. 자료 제출 요청 공문 패턴

자료 제출 요청 문장:

```text
[활용 목적]과 관련하여 [자료 요청 항목] 확인이 필요하오니, [대상 기관/부서]에서는 보유 여부를 확인한 후 제출하여 주시기 바랍니다.
```

제출자료 범위 안내:

```text
제출 대상 자료는 [자료 요청 항목]에 한정하며, 세부 범위가 불명확한 경우 [확인 필요]로 표시합니다.
```

기한 내 제출 요청:

```text
업무 추진 일정을 고려하여 [제출기한]까지 [제출방법]으로 제출하여 주시기 바랍니다.
```

개인정보 제외 안내:

```text
[개인정보 제외 안내 문구]에 따라 실제 개인정보, 민감정보, 내부 운영정보는 제출자료에서 제외하여 주시기 바랍니다.
```

문의처 안내:

```text
문의사항은 [담당부서] 또는 [공용 연락처]로 확인하여 주시기 바랍니다.
```

### 1-3. 협조 요청 공문 패턴

협조 요청 목적:

```text
[사업명] 또는 [행사명]의 원활한 추진을 위하여 [대상 기관/부서]의 협조가 필요한 사항을 안내드립니다.
```

요청사항 정리:

```text
협조 요청 사항은 [요청사항 확인 필요]이며, 세부 내용은 [확인 필요]로 구분하여 검토합니다.
```

회신 요청:

```text
협조 가능 여부와 추가 확인 사항은 [제출기한]까지 [제출방법]으로 회신하여 주시기 바랍니다.
```

담당부서 문의 안내:

```text
세부 조정이 필요한 경우 [담당부서]와 사전 협의하여 주시기 바랍니다.
```

내부 검토 후 결정 문구:

```text
최종 추진 여부와 세부 방식은 내부 검토 후 결정되며, 현재 문안은 사전 협의용 초안입니다.
```

### 1-4. 회신 요청 공문 패턴

검토 결과 회신 요청:

```text
[자료 요청 항목]에 대한 검토 결과 또는 확인 의견을 회신하여 주시기 바랍니다.
```

회신기한 안내:

```text
회신 기한은 [제출기한]으로 표시하며, 기한 조정이 필요한 경우 [담당부서]와 협의합니다.
```

회신방법 안내:

```text
회신은 [제출방법]을 통해 제출하며, 첨부자료가 있는 경우 개인정보와 내부 운영정보 포함 여부를 사전 확인합니다.
```

불필요한 개인정보 제외 안내:

```text
회신 내용에는 업무 처리에 불필요한 개인정보, 민감정보, 사건 식별정보를 포함하지 않습니다.
```

## 2. 추진계획서 문장 패턴

### 2-1. 추진배경 패턴

환경 변화에 따른 필요성:

```text
[업무 발생 배경]에 따라 [사업명] 추진 필요성이 제기되어 관련 절차와 추진 방향을 검토하고자 합니다.
```

현장 수요에 따른 필요성:

```text
[대상 기관/부서]의 현장 수요를 반영하여 [조사 대상] 또는 [자료 요청 항목]에 대한 체계적인 관리 방안을 검토하고자 합니다.
```

정책 방향에 따른 필요성:

```text
[활용 목적]에 부합하는 업무 추진을 위해 [사업명]의 기본 방향과 추진 절차를 정리하고자 합니다.
```

기존 문제점에 따른 추진 필요성:

```text
[확인 필요]로 표시된 기존 업무상 한계를 검토하고, 개선이 필요한 항목을 중심으로 추진 방안을 마련하고자 합니다.
```

### 2-2. 추진목적 패턴

사업 목적:

```text
본 계획은 [사업명] 추진을 통해 [활용 목적]을 달성하기 위한 기본 절차를 정리하는 데 목적이 있습니다.
```

대상자 편의 증진:

```text
[대상]의 이용 편의가 향상될 수 있도록 관련 절차와 안내 방식을 검토하고자 합니다.
```

업무 효율성 제고:

```text
반복되는 업무 절차를 정리하여 [담당부서]의 업무 효율성을 높이는 데 기여하고자 합니다.
```

서비스 접근성 향상:

```text
[대상]이 필요한 정보를 보다 쉽게 확인할 수 있도록 안내 체계 개선을 도모하고자 합니다.
```

안전관리 강화:

```text
[조사 대상]과 관련된 확인 절차를 정비하여 안전관리 기반을 강화하고자 합니다.
```

### 2-3. 추진개요 패턴

| 항목 | 내용 |
|---|---|
| 사업명 | [확인 필요] |
| 추진기간 | [추진기간 확인 필요] |
| 추진대상 | [확인 필요] |
| 추진방식 | [확인 필요] |
| 소요예산 | [소요예산 확인 필요] |
| 담당부서 | [담당부서] |

### 2-4. 세부 추진계획 패턴

사전 준비:

```text
사전 준비 단계에서는 [자료 요청 항목]을 확인하고, 필요한 협의 사항을 [확인 필요]로 정리합니다.
```

대상자 모집:

```text
대상자 모집 또는 참여 안내가 필요한 경우, 실제 개인정보를 수집하지 않는 범위에서 안내 문안을 작성합니다.
```

관계기관 협의:

```text
관계기관 협의가 필요한 사항은 [대상 기관/부서]와 사전 검토하되, 확정되지 않은 내용은 단정하지 않습니다.
```

운영 및 실행:

```text
운영 단계에서는 [추진방식 확인 필요]에 따라 진행하며, 세부 일정과 담당자는 [확인 필요]로 표시합니다.
```

결과 정리 및 보고:

```text
추진 결과는 [활용 목적]에 맞게 정리하되, 실제 참여자 정보, 실제 수치, 내부 운영정보는 포함하지 않습니다.
```

### 2-5. 추진일정 패턴

| 단계 | 주요 내용 | 일정 |
|---|---|---|
| 사전 준비 | [자료 요청 항목] 확인 | [확인 필요] |
| 협의 | [대상 기관/부서] 협의 | [확인 필요] |
| 실행 | [추진방식 확인 필요]에 따른 운영 | [확인 필요] |
| 정리 | 결과 정리 및 보고 | [확인 필요] |

일정이 확정되지 않은 경우 실제 날짜를 만들지 않고 [확인 필요]로 표시합니다.

### 2-6. 소요예산 패턴

예산 미확정 안내:

```text
소요예산은 [소요예산 확인 필요]로 표시하며, 실제 금액은 임의로 작성하지 않습니다.
```

항목 placeholder:

| 항목 | 산출 기준 | 금액 |
|---|---|---|
| [예산 항목] | [산출 기준 확인 필요] | [소요예산 확인 필요] |
| [예산 항목] | [산출 기준 확인 필요] | [소요예산 확인 필요] |

확인 필요 문구:

```text
예산 편성 가능 여부와 집행 기준은 관련 지침과 [담당부서] 확인 후 작성합니다.
```

### 2-7. 기대효과 패턴

```text
[사업명] 추진을 통해 [활용 목적] 달성에 기여할 것으로 보입니다.
```

```text
[대상]의 이용 편의와 업무 처리의 일관성 향상에 도움이 될 것으로 기대됩니다.
```

```text
관련 절차를 정리함으로써 향후 [후속 조치 확인 필요] 추진 기반을 마련하고자 합니다.
```

## 3. 결과보고서 문장 패턴

### 3-1. 추진결과 요약 패턴

```text
[사업명] 추진 결과, [주요 확인 사항]을 중심으로 진행 경과를 정리하고 향후 [활용 목적]에 참고하고자 합니다.
```

```text
본 보고서는 [추진기간 확인 필요] 동안의 추진 내용을 일반화하여 정리한 초안이며, 실제 성과 수치는 [확인 필요]로 표시합니다.
```

### 3-2. 주요성과 정리 패턴

```text
주요 성과는 [성과 항목 확인 필요]을 중심으로 정리하되, 실제 참여 인원, 실제 예산 집행액, 실제 기관별 결과는 포함하지 않습니다.
```

```text
[사업명] 추진을 통해 [활용 목적] 달성에 일부 기여한 것으로 보이며, 세부 평가는 [확인 필요]로 둡니다.
```

### 3-3. 문제점 및 개선사항 패턴

```text
추진 과정에서 확인된 개선 필요 사항은 [개선사항 확인 필요]로 정리하고, 세부 원인과 조치 방향은 추가 검토가 필요합니다.
```

```text
개선사항은 특정 개인, 기관, 업체를 지칭하지 않고 업무 절차와 구조 중심으로 작성합니다.
```

### 3-4. 향후계획 패턴

```text
향후 [후속 조치 확인 필요]를 검토하고, 필요한 경우 [대상 기관/부서]와 협의하여 세부 계획을 보완합니다.
```

```text
추가 추진 여부는 내부 검토 후 결정하며, 확정되지 않은 일정과 예산은 [확인 필요]로 표시합니다.
```

### 3-5. 결과보고 유의사항

- 실제 성과 수치, 참여 인원, 예산 집행액을 임의 생성하지 않습니다.
- 실제 기관별 결과, 실제 담당자 의견, 실제 내부 검토의견을 포함하지 않습니다.
- 확인되지 않은 내용은 [확인 필요]로 표시합니다.

## 4. 검토보고서 문장 패턴

### 4-1. 검토배경 패턴


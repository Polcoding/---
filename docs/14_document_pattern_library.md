# 문서 패턴 라이브러리

## 목적

공공기관 행정문서에서 반복적으로 쓰이는 문서 구조를 정리합니다. 이 문서는 원문 보관용이 아니라 패턴 정의용입니다.

## 공문 패턴

```text
문서번호: [문서번호 확인 필요]
시행일자: [YYYY.MM.DD. 확인 필요]
수    신: [수신처 확인 필요]
참    조: [참조 확인 필요]
제    목: [제목 확인 필요]

1. [추진 배경 또는 목적]

2. [요청 사항]

  가. [세부 항목]
  나. [세부 항목]
  다. [세부 항목]

3. [제출 또는 협조 요청]

붙임  1. [문서명] 1부.
      2. [문서명] 1부.  끝.
```

## 조사표 패턴

| 구분 | 점검 항목 | 작성 기준 | 확인 필요 사항 | 비고 |
|---|---|---|---|---|
| [기관 A] | [점검 항목 확인 필요] | [작성 기준 확인 필요] | [확인 필요] | [비고] |

## 기획서 패턴

1. 추진배경
2. 추진목적
3. 추진개요
4. 주요내용
5. 세부 추진계획
6. 추진일정
7. 소요예산
8. 기대효과
9. 검토사항
10. 향후계획

## 업체 메일 패턴

```text
제목: [업무명 확인 필요] 관련 자료/견적 요청의 건

안녕하세요.
[기관 A] [담당부서 확인 필요]입니다.

[업무 목적]과 관련하여 사전 검토를 위한 자료를 요청드립니다.

본 요청은 사전 검토를 위한 자료 요청이며, 계약 또는 업체 선정이 확정된 사항은 아닙니다.
회신 시 개인정보, 참석자 명단, 내부자료 또는 민감정보가 포함된 자료는 제외하여 주시기 바랍니다.

회신기한: [확인 필요]
문의처: [담당부서 공용 연락처] / [공용 이메일]
```

## 민원 외부 전달문 패턴

```text
[민원 접수 시기]에 제기된 [비식별 민원 요지]와 관련하여, 사실관계 확인을 위한 자료 검토를 요청드립니다.

붙임 자료는 비식별 처리된 민원 요지 자료로 한정하며, 민원인 성명, 연락처, 주소, 민원번호, 접수번호 등 식별 가능 정보는 포함하지 않습니다.
```

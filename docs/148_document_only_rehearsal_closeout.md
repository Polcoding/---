# 문서 기반 리허설 Closeout

## 목적

실제 HWPX 파일 없이 진행한 사용자 quick start와 리허설 문서 묶음을 하나의 문서 산출물 단위로 확정합니다.

이 문서는 실제 업무용 HWPX 파일 생성, 실제 HWPX 열람 결과, 실제 업무값 입력을 기록하지 않습니다.

## 확정한 것

| 항목 | 판단 |
|---|---|
| 문서만으로 가능한 리허설 | 가능 |
| 실제값 제외 안내 | 유지 |
| 역할 구분 | `[사용자 확인 필요]`, `[Codex 처리 가능]`, `[보류]` 유지 |
| HWPX 열람 필요 지점 | 사용자 확인으로 분리 |
| 실제 HWPX 수동 preview | 아직 보류 |
| 외부 연동 | 보류 |

## 아직 확정하지 않는 것

아래 항목은 실제 HWPX 파일과 한컴 열람이 필요하므로 확정하지 않습니다.

- HWPX 파일 열림
- 글자 겹침 여부
- 항목 순서 적정성
- 줄바꿈과 문단 배치 적정성
- 표 폭, 여백, 행 높이 적정성
- 남은 `{{placeholder}}` 여부
- 기관 표준 글꼴, 자간, 줄간격, 문단 간격 값

## 다음 단계 판단

현재는 실제 HWPX 수동 preview로 넘어가지 않습니다.

수동 preview 재개 조건:

1. 사용자가 저장소 밖 비식별 작업 복사본을 준비했다고 명시
2. 실제 원문, 개인정보, 기관명, 문서번호, 실제 표 데이터 제거 확인
3. GitHub Desktop Changes에 실제 HWPX 원본과 Git 추적 대상 output 파일이 보이지 않음
4. 사용자가 한컴에서 열람 검수를 진행할 수 있음
5. 검수 범위가 표 내부 실제값이 아니라 글자 겹침, 줄바꿈, 표 폭, 여백임

## push 판단

이 묶음은 문서 산출물로 정리된 상태입니다.

근거:

- 새 HWPX 파일을 생성하지 않음
- 실제 HWPX 열람 결과를 기록하지 않음
- 실제 원문, 개인정보, 기관명, 문서번호를 추가하지 않음
- 표 데이터 자동화를 진행하지 않음
- 외부 연동을 진행하지 않음
- 사용자가 직접 확인할 지점과 Codex가 처리할 지점을 분리함

다만 작은 문서 묶음 하나만으로 멈추지 않고, 이후 관련 검증 결과와 함께 묶어서 판단합니다.

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- Email, API, Make.com 연동 없음

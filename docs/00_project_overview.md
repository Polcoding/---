# 프로젝트 개요

## 문제 정의

공공기관 실무자는 짧은 업무 지시, 회의 메모, 자료 요청 내용을 바탕으로 공문, 기획서, 검토자료, 조사표, 업체 연락 메일을 반복적으로 작성합니다. 이 과정에서 문서 구조와 문체는 반복되지만, 실제 내용에는 개인정보, 내부 운영정보, 민감자료가 섞일 수 있습니다.

이 프로젝트는 AI가 문서 초안을 빠르게 만들도록 돕되, 민감정보 보호와 사람의 최종 승인을 전제로 합니다.

## 목표 사용자

- 행정문서 초안 작성 시간을 줄이고 싶은 실무자
- 보안 기준을 지키면서 AI 테스트판을 만들고 싶은 개발자
- 향후 자동화 시스템을 단계적으로 구현하려는 담당자

## 입력 방식

초기 단계의 입력은 짧은 자연어 지시입니다.

예시는 다음처럼 placeholder만 사용합니다.

- `[기관 A] 대상 조사 요청 공문 초안 작성`
- `[업체 A]에 사전 견적 요청 메일 초안 작성`
- `[사업명 확인 필요] 추진계획서 목차와 초안 작성`

## 출력 방식

AI는 다음과 같은 초안만 생성합니다.

- 공문 초안
- 기획서 초안
- 검토자료 초안
- 조사표 양식
- 작성요령
- 업체 연락 메일 초안
- 발송 전 체크리스트

모든 출력은 사용자가 검토하고 수정한 뒤 내부 절차에 따라 승인해야 합니다.

## 사람이 승인하는 구조

AI는 행정 실무자의 판단을 대체하지 않습니다. AI가 만든 결과는 초안이며, 실제 발송ㆍ결재ㆍ계약ㆍ예산 집행ㆍ민원 처리 결론은 담당자가 판단합니다.

필수 검토자는 다음 항목을 확인합니다.

- 사실관계
- 개인정보 포함 여부
- 내부 승인 필요 여부
- 첨부파일 적정성
- 기관 문서 기준과 문체
- 외부 발송 가능 여부

## AI에게 직접 발송 권한을 주면 안 되는 이유

외부 발송은 문서 내용, 수신처, 첨부파일, 개인정보 포함 여부를 모두 확인해야 하는 업무입니다. AI에게 직접 발송 권한을 주면 잘못된 수신처, 확인되지 않은 사실, 민감정보 포함 문서가 외부로 나갈 위험이 있습니다.

따라서 1단계에서는 초안 생성까지만 허용하고, 발송은 사람이 담당합니다.

## 기존 공문 원문을 학습시키면 안 되는 이유

기존 공문 원문에는 실제 기관명, 문서번호, 결재 흐름, 내부 운영정보, 민원 정보, 업체 정보가 포함될 수 있습니다. 원문을 그대로 Custom GPT Knowledge에 업로드하면 불필요한 민감정보가 외부 시스템에 입력될 수 있습니다.

이 프로젝트에서 가져올 것은 원문 내용이 아니라 서식ㆍ문체ㆍ구조 기준입니다.

## 문서 서식은 템플릿 엔진이 담당해야 하는 이유

폰트, 자간, 줄간격, 표 양식, 여백 같은 세부 서식은 AI가 안정적으로 맞추기 어렵습니다. AI는 문서의 의미 구조와 초안을 만들고, 실제 서식은 템플릿과 렌더러가 처리해야 합니다.

현재 저장소에서는 placeholder 기반 로컬 HWPX 렌더러 PoC를 통해 원페이지 보고서, 추진계획서, 결과보고서, 검토보고서 4종의 치환과 한컴 수동 열람 검수를 완료했습니다. 다만 실제 기관 양식 원본을 저장소에 추가하거나 운영 자동화를 수행한 것은 아닙니다.

HWPX 보고서 안의 표는 현재 단계에서 표 틀과 배치 검수 대상으로만 다룹니다. 표 내부 데이터와 계산은 향후 Excel/한셀 연동 후보로 분리합니다.

비식별 HWPX 작업 복사본이 없으면 실제 양식 수동 preview 결과를 임의로 만들지 않고 보류합니다.

## 사용자 입력 역할 구분

사용자에게 입력을 요청할 때는 실제값을 임의로 채우지 않고 역할을 구분합니다.

- `[사용자 확인 필요]`: 사람이 직접 판단하거나 확인해야 하는 값
- `[Codex 처리 가능]`: 비식별 placeholder 입력을 바탕으로 구조화 가능한 항목
- `[보류]`: 현재 단계에서 자동화하지 않는 항목

## 전체 아키텍처 도식

```text
사용자 입력
→ 입력 정규화
→ 보안 필터
→ 업무 유형 분류
→ AI 초안 생성
→ 템플릿 렌더링
→ 사용자 검토
→ 사람 승인
```

## 현재 범위

현재 저장소는 Phase 1 문서화, Phase 2 최소 PoC, Phase 3 안전 게이트 문서화, Phase 4 문서 템플릿 안정화 진입 판단, 저장소 밖 실제 양식 후보 수동 절차 재정렬, local template Git 제외 반복 검증 기준, Phase 4 통합 점검, 실제 양식 수동 리허설 사용자 확인 패킷 정리, 사용자 준비 확인 완료 기록, HWPX 일원화 유지와 표 데이터 Excel/한셀 연동 후보 분리 결정, 비식별 HWPX 작업 복사본 없음에 따른 수동 리허설 보류 기록, HWPX 보고서와 Excel/한셀 표 데이터 역할 분리 범위 정리, HWPX 보고서 4종 사용자 입력 템플릿의 표 데이터 후보 표시 기준 정리, 보고서 샘플 JSON 4종의 표 데이터 후보 직접 반영 보류 결정, renderer와 normalizer 안내 문서의 표 데이터 후보 오해 가능성 점검, 최소 demo 실행 결과 요약, 사용자 입력 역할 구분의 반복 운영 로그 연결, 사용자 운영 A-to-Z 안내와 비식별 요청 예시 정리, 사용자 안내 3종 통합 점검, 사용자 quick start 정리, quick start 리허설 경계 정리, 사용자 안내 묶음 closeout, 수동 preview 또는 리허설 유지 판단, 문서 기반 사용자 리허설 결과, 문서 기반 리허설 closeout, 문서 기반 리허설 hold 상태 정리까지 완료한 상태입니다.

## 다음 단계

다음 단계는 실제 API 연동이나 실제 양식 투입이 아니라, 문서 기반 hold 상태를 보안 검수와 Git 제외 검증 후 push 단위로 확정하는 것입니다.

이미 추가된 `placeholder_confirmed_values` read-only helper는 `missing_fields` 생성 결과나 routing 결과를 바꾸지 않습니다.

실제 기관 양식 원본은 저장소 밖에서만 다루며, 저장소에는 절차 문서와 placeholder 기반 샘플만 남깁니다.

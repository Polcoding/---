# 문서 기반 사용자 리허설 결과

## 목적

실제 HWPX 파일을 생성하거나 열지 않고, 사용자 quick start와 리허설 체크리스트만으로 현재 단계에서 무엇을 확인할 수 있는지 기록합니다.

이 문서는 실제 업무값 입력, 실제 HWPX 생성, 실제 HWPX 열람 결과 기록이 아닙니다.

## 리허설 기준 문서

- `docs/143_user_quick_start.md`
- `docs/144_quick_start_rehearsal_boundary.md`
- `checklists/user_operation_atoz_rehearsal_checklist.md`
- `docs/141_user_rehearsal_prompt_examples.md`
- `docs/146_next_manual_preview_or_rehearsal_decision.md`

## 리허설 범위

| 항목 | 결과 | 비고 |
|---|---|---|
| 처음 열 문서 확인 | 문서 기준 확인 | `CURRENT_STATUS.md`와 quick start 기준 확인 가능 |
| 문서 유형 선택 흐름 | 문서 기준 확인 | 보고서 4종 예시 유지 |
| 실제값 제외 안내 | 문서 기준 확인 | 실제 원문, 개인정보, 기관명, 문서번호 제외 기준 유지 |
| 역할 구분 | 문서 기준 확인 | `[사용자 확인 필요]`, `[Codex 처리 가능]`, `[보류]` 유지 |
| HWPX 열람 필요 지점 분리 | 문서 기준 확인 | 한컴 확인 항목은 사용자 확인으로 남김 |
| 표 데이터 자동화 보류 | 문서 기준 확인 | 실제 표 데이터 입력 요구 없음 |
| 외부 연동 보류 | 문서 기준 확인 | Email/API/Make.com 연동 없음 |
| 실제 HWPX preview | 보류 | 비식별 작업 복사본 준비와 사용자 한컴 확인 필요 |

## 문서만으로 가능한 확인

- 사용자가 처음 볼 문서 순서 확인
- 보고서 4종 비식별 요청 예시 확인
- 실제값을 넣지 않는 입력 방식 확인
- HWPX 열람이 필요한 지점과 아닌 지점 구분
- GitHub Desktop Changes에서 HWPX/output이 보이면 중단해야 함을 확인

## 사용자 확인으로 남기는 항목

아래 항목은 실제 HWPX 파일과 한컴 열람이 필요하므로 문서 기반 리허설에서 확정하지 않습니다.

- HWPX 파일 열림
- 글자 겹침
- 항목 순서
- 줄바꿈과 문단 배치
- 표 폭, 여백, 행 높이
- 남은 `{{placeholder}}`
- bullet 또는 번호 표기 일관성
- 기관 표준 글꼴, 자간, 줄간격, 문단 간격 값

## 현재 판단

문서 기반 사용자 리허설은 현재 단계에서 가능합니다.

실제 HWPX 수동 preview는 아직 재개하지 않습니다.

수동 preview는 사용자가 저장소 밖 비식별 작업 복사본을 준비하고, 한컴에서 파일 열람 검수를 요청할 때 재개합니다.

## 다음 추천

다음 추천은 문서 기반 리허설 묶음을 push 가능한 단위로 확정하는 것입니다.

push 이후에는 실제 HWPX 없이 계속 문서 기반 리허설을 유지하거나, 사용자가 비식별 작업 복사본을 준비했다고 명시했을 때 수동 preview 준비 문서로 넘어갑니다.

## 보안 검수

- 실제 개인정보 추가 없음
- 실제 기관명, 업체명, 담당자명 추가 없음
- 실제 문서번호, 민원번호, 사건번호 추가 없음
- 실제 날짜, 예산, 실적 수치 추가 없음
- 실제 표 데이터, 물품명, 수량, 금액, 대상 목록 추가 없음
- 실제 공문 원문 또는 보고서 원문 추가 없음
- 실제 HWPX/HWP 원본 파일 추가 없음
- Email, API, Make.com 연동 없음

# 보안 정책

## 목적

AI가 처리할 수 있는 업무와 처리하면 안 되는 업무를 명확히 구분합니다.

## 처리 가능 업무

다음 업무는 비식별화된 지시와 placeholder만 포함된 경우 처리할 수 있습니다.

- 공문 초안 작성
- 기획서 초안 작성
- 조사표 양식 작성
- 작성요령 작성
- 업체 연락 메일 초안 작성
- 회의자료 구조 정리
- 자료조사 요청 문안 정리
- 발송 전 체크리스트 작성

## 처리 금지 업무

다음 요청은 처리하지 않거나 비식별화 후 재요청을 안내합니다.

- 개인정보가 포함된 문서를 그대로 작성하는 요청
- 대외비 또는 비공개 문서 내용을 처리하는 요청
- 내부망 자료를 외부 AI에 입력하도록 하는 요청
- 실제 차량번호, 실제 관서별 운영정보, 실제 장비 수량을 포함한 요청
- 업체 선정, 계약 확정, 예산 확보를 단정하는 문구 작성 요청
- 법률적으로 문제가 없다고 단정하는 요청
- 실제 발송, 결재, 계약, 예산 집행을 자동화하는 요청

## 위험도 분류

| 위험도 | 기준 | 처리 방식 |
|---|---|---|
| low | 공개 가능한 일반 업무 지시, 개인정보 없음, 내부자료 없음 | 초안 작성 가능 |
| medium | 내부 운영정보 가능성이 있으나 실제 세부자료는 없고 문서 틀만 요청 | 공문ㆍ양식ㆍ작성요령 틀만 작성 |
| high | 개인정보, 대외비, 실제 내부 현황자료, 계정, 비밀번호, 내부망 정보, 실제 민원인ㆍ직원ㆍ업체 민감정보 포함 | 처리 제한, 비식별화 후 재요청 안내 |

## 외부 발송 금지 원칙

AI는 외부 발송을 수행하지 않습니다. 외부 발송 문안은 항상 초안으로 작성하며, 담당자 검토와 내부 승인 후 사용해야 합니다.

## 사람 최종 승인 원칙

최종 문서는 사람이 검토합니다. 특히 다음 항목은 사람이 확인해야 합니다.

- 사실관계
- 수신처
- 문서번호
- 시행일자
- 첨부파일
- 개인정보 포함 여부
- 기관 내부 승인 필요 여부

# HWPX 보고서 템플릿 자동화 우선순위

## 목적

이 문서는 본 프로젝트의 핵심 자동화 대상을 HWPX 기반 보고서 템플릿 자동화로 재정의합니다.

실무에서 가장 자주 작성하는 문서는 원페이지 보고서, 추진계획서, 결과보고서이며, 기존 HWP/HWPX 정형 양식을 수정하여 작성하는 방식이 일반적입니다.

따라서 향후 PoC와 고도화는 보고서 템플릿 자동화를 우선합니다.

## 자동화 우선순위

| 우선순위 | 대상 | 출력 형식 | 비고 |
|---|---|---|---|
| 1 | 원페이지 보고서 | HWPX | 최우선 자동화 대상 |
| 2 | 추진계획서 | HWPX | 계획 수립 문서 |
| 3 | 결과보고서 | HWPX | 기존 계획 대비 결과 정리 |
| 4 | 검토보고서 | HWPX | 내부 검토자료 |
| 5 | 조사표 | XLSX | 보조 자동화 대상 |
| 6 | 업체 메일 | Email | 보조 초안 작성 |

## 원페이지 보고서 자동화 기준

원페이지 보고서는 다음 구조를 기본 후보로 둡니다.

- 제목
- 보고 개요
- 추진 배경
- 주요 내용
- 검토 의견
- 문제점 또는 고려사항
- 향후 계획
- 조치 필요사항 또는 협조사항

문체 기준:

- 개조식 중심
- 짧은 문장
- 명사형 표현
- 불필요한 조사 최소화
- 구어체 지양
- 확정되지 않은 내용은 [확인 필요]
- 예산, 일정, 수량, 담당자는 임의 생성 금지

## 추진계획서 자동화 기준

추진계획서는 다음 구조를 기본 후보로 둡니다.

- 추진배경
- 추진목적
- 추진개요
- 세부 추진계획
- 추진일정
- 소요예산
- 기대효과
- 검토사항 또는 행정사항
- 붙임

주의:

- 예산과 일정은 임의 생성하지 않습니다.
- 담당부서와 대상자는 [확인 필요]로 표시합니다.
- 기대효과는 과장하지 않습니다.

## 결과보고서 자동화 기준

결과보고서는 기존 계획서와 연결되는 구조를 기본 후보로 둡니다.

- 추진개요
- 추진결과
- 주요성과
- 문제점 및 개선사항
- 향후계획
- 붙임

주의:

- 참여 인원, 실적 수치, 예산 집행액은 임의 생성하지 않습니다.
- 기존 추진계획서와 연결되는 항목은 가능한 한 대응 구조로 정리합니다.
- 사용자가 제공하지 않은 결과값은 [확인 필요]로 표시합니다.

## 기존 HWP/HWPX 템플릿 활용 원칙

- 원본 HWP/HWPX 양식은 직접 수정하지 않습니다.
- 복사본을 만들고 실제 내용을 제거한 뒤 placeholder를 삽입합니다.
- 실제 기관명, 문서번호, 결재선, 직인, 로고는 사용하지 않습니다.
- placeholder 기반 테스트 템플릿은 처음에는 로컬에서만 관리합니다.
- Git에 HWP/HWPX 파일을 추가하려면 별도 검수와 명시적 승인이 필요합니다.

## 공문서 작성 요령 PDF 반영 계획

향후 사용자가 공문서 작성 요령 PDF를 제공하면 다음 방식으로 반영합니다.

- PDF 원문을 그대로 학습시키지 않습니다.
- 작성 원칙, 문장 패턴, 번호체계, 지양 표현, 권장 표현만 추출합니다.
- 긴 원문 인용을 피하고 요약ㆍ패턴화합니다.
- 보고서 문체, 계획서 문체, 결과보고서 문체, 공문체를 분리하여 반영합니다.
- 민감정보나 내부자료가 포함된 PDF는 외부 AI 처리 대상에서 제외합니다.

## Step 결론

향후 HWPX 자동화의 중심은 공문보다 보고서 템플릿 자동화입니다.

Email 렌더러는 보조 기능으로 유지하되, 협조 요청 메일은 정중한 실무 메일체를 사용합니다.

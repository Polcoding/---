# 다음 단계 판단: 수동 preview 또는 리허설 유지

## 목적

사용자 안내 묶음 closeout 이후 다음 단계가 실제 HWPX 수동 preview인지, 실제 HWPX 없이 문서 기반 리허설을 유지할지 판단합니다.

이 문서는 실제 업무용 HWPX 파일 생성 지시가 아닙니다. 실제 원문, 개인정보, 기관명, 문서번호, 실제 표 데이터, 실제 HWPX/HWP 원본을 기록하지 않습니다.

## 현재 판단

현재 단계에서는 실제 HWPX 수동 preview로 넘어가지 않습니다.

추천 방향은 실제 HWPX 없이 사용자 리허설을 유지하는 것입니다.

이유:

- 비식별 작업 복사본이 새로 준비되었다는 확인이 없음
- 실제 기관 HWPX/HWP 원본을 저장소에 넣지 않는 원칙 유지 필요
- 사용자 안내 묶음만으로도 현재 작동 범위와 중단 지점을 확인 가능
- HWPX 열람 검수는 사람이 한컴에서 직접 확인해야 하는 단계로 남아 있음

## 수동 preview 진입 조건

아래 조건이 모두 충족될 때만 실제 HWPX 수동 preview를 재개합니다.

1. 사용자가 저장소 밖 비식별 작업 복사본을 준비했다고 명시
2. 실제 원문, 개인정보, 기관명, 문서번호, 실제 표 데이터가 제거됨
3. GitHub Desktop Changes에 실제 HWPX 원본과 output이 보이지 않음
4. 사용자가 한컴에서 파일 열람 검수를 진행할 수 있음
5. 검수 대상이 표 내부 실제값이 아니라 표 틀, 줄바꿈, 여백, 글자 겹침인지 확인됨

## 문서 기반 리허설 유지 조건

아래 중 하나라도 해당하면 실제 HWPX 수동 preview는 보류하고 문서 기반 리허설을 유지합니다.

- 비식별 작업 복사본 준비 여부가 확인되지 않음
- 실제 HWPX/HWP 원본 위치 또는 Git 제외 상태가 불명확함
- 실제 원문, 개인정보, 기관명, 문서번호가 포함될 가능성이 있음
- 사용자가 한컴 열람 확인을 진행할 수 없는 상태임
- 표 내부 데이터 자동화 요구가 섞임
- Email/API/Make.com 등 외부 연동 요구가 섞임
